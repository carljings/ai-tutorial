扩散模型
从噪声中生长出图像的生成艺术

如果说 GAN 是两个网络的博弈，那么扩散模型就是一场"先破坏再重建"的炼金术。本章带你从 DDPM 的前向加噪与反向去噪出发，拆解 Stable Diffusion 的潜空间架构，并用 Hugging Face diffusers 库的 5 行代码亲手生成你的第一张图。

阅读约 22 分钟 难度 ★★★★

CFG 引导与 DDIM 加速

diffusion 生成质量提升的两个核心技巧：CFG（Classifier-Free Guidance）用引导强度控制'多听 prompt'，DDIM 用确定性跳步采样把步数从 50 降到 20。理解它们，你就能在质量和速度间精准调权。

# Classifier-Free Guidance: 混合条件/无条件预测
import torch
def cfg_step(model, x, t, cond_emb, uncond_emb, guidance_scale=7.5):
    """guidance_scale 越大, 越听从 prompt(但多样性下降)。"""
    eps_cond   = model(x, t, cond_emb)      # 条件预测
    eps_uncond = model(x, t, uncond_emb)    # 无条件预测
    # CFG 公式: eps = uncond + scale*(cond - uncond)
    return eps_uncond + guidance_scale * (eps_cond - eps_uncond)
# 演示 guidance_scale 的影响(用随机张量模拟)
torch.manual_seed(0)
x = torch.randn(1, 4, 32, 32)
for s in [1.0, 3.0, 7.5, 15.0]:
    # s=1 等于不用引导; s=7.5 是 SD 默认; s 过大会过饱和
    fake_cond = torch.randn_like(x)
    fake_uncond = torch.randn_like(x) * 0.3
    eps = fake_uncond + s * (fake_cond - fake_uncond)
    print(f"guidance={s:5.1f}  eps范数={eps.norm():.2f}  "
          f"prompt影响={ (s/(s+1))*100:.0f}%")
# DDIM 加速: set_timesteps(20) 配合 DDIMScheduler 即可,
# 它跳过中间步、用确定性更新, 20步接近50步DDPM质量。

★ DDPM 反向去噪采样循环详解

前面讲了 DDPM 的前向加噪（把图像逐步加高斯噪声变成纯噪声）和反向去噪（从纯噪声逐步还原成图像）的数学原理，但原理要变成能跑的代码，关键在采样循环。采样循环是 diffusion 模型推理的全部：从一张纯高斯噪声出发，循环 T 步，每一步用 UNet 预测当前噪声、再用调度器（scheduler）把噪声减掉一点点，最后得到一张干净图像。理解这个循环，你就掌握了所有 diffusion 模型（DDPM、DDIM、Stable Diffusion）推理的共同骨架——它们只是换了调度器或换了 UNet 结构。

我们用 Hugging Face 的 diffusers 库来实现，并刻意把采样循环手动展开，这样你能看清每一步在干什么。先加载一个预训练的小型 DDPM（在 CIFAR-10 上训练的 32×32 模型）和它的调度器。关键概念：scheduler.set_timesteps(50) 把去噪步数从训练时的 1000 步压缩到 50 步——这是推理加速的核心，步数越少越快但质量略降，50 步通常是质量与速度的甜点。

# -*- coding: utf-8 -*-
"""DDPM 反向去噪采样循环：从纯噪声出发，逐步去噪生成图像。
这是 diffusion 模型推理的核心。下面用 diffusers 库实现，
并手动展开采样循环以便理解每一步。
"""
import torch
from diffusers import DDPMScheduler, UNet2DModel

# 1) 加载一个预训练 UNet + 调度器(以小尺寸示意)
model = UNet2DModel.from_pretrained("google/ddpm-cifar10-32")
model.eval()
scheduler = DDPMScheduler.from_pretrained("google/ddpm-cifar10-32")
scheduler.set_timesteps(50)          # 采样 50 步(可调)

手动展开的采样循环里，有三件事值得看懂。第一，起点是纯噪声 x = torch.randn(shape)，这是 diffusion 和 GAN 的本质区别——GAN 从一个隐向量出发一次性生成，diffusion 从噪声出发迭代修正。第二，UNet 预测的是噪声 eps 而不是干净图像，这是 DDPM 的参数化选择（叫 "eps-prediction"），调度器拿到 eps 后按公式 x_{t-1} = (x_t - beta_t * eps) / sqrt(1-beta_t) 做一步去噪。第三，信噪比（SNR）随步数单调上升，这是健康的采样过程的标志——如果 SNR 不升反降，说明模型预测的噪声方向错了，生成会崩坏。循环代码如下：

# 2) 手动采样循环：直观看到每一步去噪
@torch.no_grad()
def sample(model, scheduler, shape, device="cpu"):
    x = torch.randn(shape, device=device)   # 起点: 纯高斯噪声
    for i, t in enumerate(scheduler.timesteps):
        # 模型预测"当前噪声图" eps
        eps = model(x, t).sample
        # 调度器根据 eps 算出 x_{t-1} (少一点噪声)
        x = scheduler.step(eps, t, x).prev_sample
        if i % 10 == 0:
            # 打印信噪比趋势: 应单调上升(越来越像真实图)
            snr = (x.mean() / (x.std() + 1e-6)).item()
            print(f"step {i:3d} t={t:4d}  SNR≈{snr:+.2f}")
    return x

img = sample(model, scheduler, (1, 3, 32, 32))
print("生成图 shape:", img.shape, "值域:", img.min().item(), img.max().item())

掌握这个循环，你就能玩转各种 diffusion 进阶技巧。换调度器就换采样策略：把 DDPMScheduler 换成 DDIMScheduler 能用更少步数（20 步）达到相近质量；换成 EulerDiscreteScheduler 是 Stable Diffusion XL 的默认选择，配合流匹配（flow matching）效果更好。改参数化方式能改变模型行为：eps-prediction 通用稳定、v-prediction 在高噪声区更稳、flow-matching 是 2024 年后的新主流。理解了这些，你再看任何扩散模型的代码都不会陌生——它们都是在这个采样循环的基础上做变体。最后给一个调优经验：如果生成图糊，先尝试增加采样步数到 100；如果有噪点，换 DDIM；如果想要多样性强，调高起始噪声的方差。这些手感都建立在你真正读懂这个循环之上。

★ 为什么扩散模型改变了世界

2020 年之前，图像生成的主流是 GAN（生成对抗网络）。GAN 生成质量高，但训练极不稳定——模式崩溃、梯度消失、对抗失衡，调参几乎是一门玄学。直到 2020 年 UC Berkeley 的 Jonathan Ho 等人发表了 DDPM（Denoising Diffusion Probabilistic Models），用一种极其优雅的方式重新定义了生成模型。

扩散模型的灵感来自热力学：一滴墨水滴入清水，会逐渐扩散直至均匀分布——这是一个不可逆的熵增过程。DDPM 的核心思想是：如果我能学会"逆转"这个扩散过程，我就能从均匀分布的噪声中，一步步"凝固"出一张清晰的图像。

这个看似简单的想法，最终催生了 DALL·E 2、Stable Diffusion、Midjourney 等轰动全球的产品。扩散模型不仅训练稳定、生成多样性好，更重要的是它提供了一个可控的生成框架——你可以用文本、草图、深度图来精确引导生成过程，这是 GAN 难以做到的。

★ 前向过程：给图像加噪的数学之美

扩散过程分为两步：前向加噪和反向去噪。前向过程非常简单——给定一张原始图像 x_0，我们在 T 个时间步内逐步添加高斯噪声，直到它变成纯噪声 x_T。

每一步加噪的公式是：x_t = √(1-β_t) · x_{t-1} + √(β_t) · ε，其中 β_t 是预设的噪声调度参数（从 0.0001 线性增长到 0.02），ε 是标准正态分布噪声。这个公式有一个极其重要的性质——它允许我们直接从 x_0 跳跃到任意时间步 t 的噪声状态，而不必一步步迭代：

import torch

def forward_diffusion_sample(x_0, t, betas):
    """直接从 x_0 采样任意时间步 t 的加噪版本"""
    alphas = 1.0 - betas
    alphas_cumprod = torch.cumprod(alphas, dim=0)
    
    sqrt_alpha_prod = alphas_cumprod[t] ** 0.5
    sqrt_one_minus_alpha_prod = (1 - alphas_cumprod[t]) ** 0.5
    
    noise = torch.randn_like(x_0)
    # x_t = √(ᾱ_t) · x_0 + √(1-ᾱ_t) · ε
    return sqrt_alpha_prod * x_0 + sqrt_one_minus_alpha_prod * noise, noise

# 模拟 1000 步的加噪调度
betas = torch.linspace(1e-4, 0.02, 1000)
x_0 = torch.randn(1, 3, 64, 64)  # 假设的干净图像
for t in [0, 100, 500, 999]:
    x_t, _ = forward_diffusion_sample(x_0, t, betas)
    print(f"t={t:4d}: 信噪比 ≈ {(alphas_cumprod[t]/(1-alphas_cumprod[t])):.1f}")

这个"直接跳跃"的性质至关重要——它让训练时可以随机采样任意时间步来训练去噪网络，大大提高了训练效率。前向过程本身不需要学习，它是一个固定的马尔可夫链。真正需要学习的是反向过程。

★ 反向过程：学习从噪声中重建图像

反向去噪是扩散模型的核心。给定 x_T（纯噪声），模型需要学会预测每一步应该减去多少噪声，最终还原出 x_0。DDPM 的作者发现，与其预测完整的均值和方差，不如让网络直接预测每一步添加的噪声 ε，然后用一个简单的公式重建图像。

训练目标极其简洁：随机采样一张图像 x_0，随机采样一个时间步 t，用前向公式加噪得到 x_t，然后让神经网络（通常是一个 U-Net）预测噪声，损失函数就是预测噪声与真实噪声的均方误差：

def train_step(model, x_0, optimizer, T=1000):
    """DDPM 的训练步骤，极其简洁"""
    batch_size = x_0.shape[0]
    
    # 1. 随机采样时间步
    t = torch.randint(0, T, (batch_size,), device=x_0.device)
    
    # 2. 前向加噪（直接跳跃到时间步 t）
    noise = torch.randn_like(x_0)
    x_t = add_noise(x_0, t, noise)  # 上一节的公式
    
    # 3. 模型预测噪声
    predicted_noise = model(x_t, t)
    
    # 4. 简单的 MSE 损失
    loss = torch.nn.functional.mse_loss(predicted_noise, noise)
    
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
    return loss.item()

采样（推理）时，从 x_T 开始，反复执行：用模型预测噪声 → 减去噪声 → 得到 x_{t-1}，循环 T 步得到最终图像。原始 DDPM 需要 1000 步采样，速度较慢；后来的 DDIM（Denoising Diffusion Implicit Models）通过跳步采样，将采样步数压缩到 20-50 步，速度提升 50 倍以上，同时保持生成质量。

★ Stable Diffusion：潜空间扩散的革命

原始 DDPM 直接在像素空间（如 512×512×3 = 786432 维）进行扩散，计算量极其庞大。Stable Diffusion（原名 Latent Diffusion）由 CompVis 团队在 2022 年提出，做了一个天才般的改进：把扩散过程搬到压缩后的"潜空间"进行。

Stable Diffusion 由三个核心组件构成：VAE 编码器/解码器把 512×512×3 的图像压缩到 64×64×4 的潜空间表示（压缩 48 倍），扩散过程在这个小空间内进行；U-Net 去噪网络负责在潜空间去噪；文本编码器（CLIP Text Encoder）把文本提示转化为条件向量，通过交叉注意力机制注入 U-Net，引导生成方向。

这种架构带来了两个巨大优势：一是显存占用从数十 GB 降到 8 GB，让普通消费级 GPU 也能运行；二是文本条件控制让"文生图"成为可能。Stable Diffusion 的开源更是引爆了整个生态——LoRA 微调、ControlNet 精确控制、图生图、Inpainting 等技术层出不穷。

# ControlNet 架构示意：在 U-Net 上附加条件控制分支
# 输入条件可以是：边缘图、深度图、人体姿态、草图等

class ControlNetBlock:
    """复制 U-Net 的编码器部分，添加零卷积连接"""
    def __init__(self, unet_encoder):
        self.copy_encoder = copy.deepcopy(unet_encoder)  # 冻结的副本
        self.zero_conv = nn.Conv2d(320, 320, kernel_size=1)
        nn.init.zeros_(self.zero_conv.weight)  # 零初始化，训练初始等价于无 ControlNet
        
    def forward(self, x, timesteps, context, control_image):
        # 控制图像通过零卷积后注入，训练时逐步学习控制能力
        control_features = self.copy_encoder(x, timesteps, context) + control_image
        return self.zero_conv(control_features)

Hugging Face 的 diffusers 库把扩散模型的使用门槛降到了最低。安装后只需几行代码就能运行 Stable Diffusion：

# 2. 输入提示词生成图像

这里有两个关键参数：guidance_scale（CFG Scale）控制文本引导的强度——值越大，图像越贴合提示词但多样性下降；值越小，模型越自由发挥但可能偏离提示。通常 7-9 是较好的平衡点。num_inference_steps 是去噪步数，20-50 步即可，更多步数收益递减。

想要更强的控制力？可以用 StableDiffusionControlNetPipeline 配合一张边缘图或姿态图来生成；想用极低显存（4 GB）？可以加载 LCM-LoRA 或 Turbo 蒸馏模型，配合 enable_model_cpu_offload() 实现极速生成。

深 扩散模型调优：采样器、步数与 CFG 实战

同一个扩散模型，换一个采样器或调一下 CFG，出图质量和速度能差几倍。很多人卡在“效果不行就加步数”，其实问题往往在采样器选择和 CFG 配比上。下面用 diffusers 给出可复现的调优实验。

采样器与 CFG 对比实验

核心代码：固定随机种子和 prompt，遍历常见采样器与 CFG 值，批量生成并对比。这是建立“直觉”最快的方式——看图比读论文直观。

import torch
from diffusers import StableDiffusionPipeline

pipe = StableDiffusionPipeline.from_pretrained(
    "runwayml/stable-diffusion-v1-5", torch_dtype=torch.float16).to("cuda")

prompt = "a cyberpunk city at night, neon, highly detailed, 8k"
samplers = {"Euler a": "EulerAncestralDiscreteScheduler",
            "DPM++ 2M": "DPMSolverMultistepScheduler",
            "DDIM": "DDIMScheduler"}

for name, sched in samplers.items():
    pipe.scheduler = getattr(__import__("diffusers", fromlist=[sched]), sched).from_config(
        pipe.scheduler.config)
    for cfg in (7.0, 11.0):
        for steps in (20, 30):
            g = torch.Generator().manual_seed(42)
            img = pipe(prompt, num_inference_steps=steps, guidance_scale=cfg, generator=g).images[0]
            img.save(f"out_{name.replace(' ','')}_{cfg}_{steps}.png")
# 经验：DPM++ 2M @ 20 步 / CFG 7 是速度与质量的甜点

跑完这组实验你会发现：Euler a 速度快但细节偏“飘”；DPM++ 2M 在 20 步就能逼近 50 步 DDIM 的质量，是目前性价比最高的默认选择；CFG 太低（<5）图像糊且不遵 prompt，太高（>15）则色彩过饱和、出现伪影。

CFG 的本质与“动态 CFG”技巧

Classifier-Free Guidance(CFG) 的本质是放大条件生成与无条件生成的差异：CFG=7 意味着输出沿“符合 prompt”方向偏移 7 倍。所以 CFG 越高越“听话”，但也越极端。一个进阶技巧是动态 CFG：去噪早期用高 CFG（如 10）快速建立结构，后期降到 5~6 收敛细节，避免色彩爆炸。许多商用 API（如 GLM 的图像接口）内部已采用类似策略。另一个常见坑是负面 prompt 配合低 CFG 无效——负面 prompt 的作用被 CFG 放大，CFG 太低时它几乎不起作用。记住：调扩散模型永远是“采样器+步数+CFG”三者联调，单看一个参数没意义。

Q1：为什么同样的 prompt，每次出图差别很大？

因为扩散模型的随机性来自初始噪声，不同 seed 生成完全不同的图。想复现就固定 seed。想多样性就调高 seed 范围。一个常见误区是以为「prompt 一样图就一样」——其实 seed、采样器、步数、CFG 任何一个变了，结果都不同。生产中必须记录全部参数才能复现和调试。另一个影响一致性的是prompt 的措辞——同义词替换可能出截然不同的图，要建立 prompt 模板规范化。

Q2：怎么让生成的角色/风格在多张图间保持一致？

纯 prompt 很难做到，要用专门技术：IP-Adapter（输入参考图，保持角色一致性）、LoRA（微调出特定风格/角色的专属权重）、ControlNet（控制构图、姿态）。组合用最有效：LoRA 锁定风格 + IP-Adapter 保持角色 + 固定 seed 稳定构图。这是 IP/品牌设计、漫画连载等场景的刚需，也是商用文生图和玩具级 demo 的分水岭。

★ 本章小结

扩散模型的核心是"加噪-去噪"的对称过程：前向过程用固定公式逐步将图像变成噪声，反向过程用 U-Net 学习从噪声中重建图像，训练目标是简单的噪声预测 MSE 损失。DDIM 采样器解决了速度问题，Latent Diffusion（Stable Diffusion）通过潜空间压缩解决了显存问题，文本条件的交叉注意力注入解决了可控生成问题。这三项创新叠加，让扩散模型成为当前图像生成的事实标准。

理解了这些原理，你就能明白为什么不同的提示词和参数会产生不同的效果，也能更好地使用 LoRA、ControlNet 等高级工具。下一章我们将转向语音领域，探索语音大模型如何将同样的"编码-解码"思想应用到音频上。
