Q/K/V 投影矩阵
为什么需要三个不同的 Wq、Wk、Wv

上一章我们推完了 Attention(Q,K,V)=softmax(QK^T/√d_k)V，但留下一个悬念：输入明明只有一个序列 X，Q、K、V 是从哪冒出来的？答案藏在三个投影矩阵 Wq、Wk、Wv 里。本章讲清为什么必须是三个不同的矩阵、它们各自学了什么、以及「让一个 token 能同时扮演提问者、被查询者、信息携带者三种角色」的精妙设计。

阅读约 26 分钟 难度 ★★★★ 前置：第 94 章

★ Q、K、V 从哪来

把 Transformer 的注意力层画成数据流图，你会看到这样的结构：

输入 X （n × d_model）
▼ 三条投影
Q = X · Wq
(d_model → d_k)
K = X · Wk
(d_model → d_k)
V = X · Wv
(d_model → d_v)
▼ 送入注意力
softmax(QK^T/√d_k) · V
图 95-1 · 一个输入 X，通过三个投影矩阵，变成 Q、K、V

关键就一句话：Q、K、V 都是输入 X 经过不同线性变换得到的。

Q = X · Wq    K = X · Wk    V = X · Wv
其中 Wq、Wk 是 d_model × d_k 的矩阵，Wv 是 d_model × d_v 的矩阵。

这三个 W 矩阵就是模型要学习的参数。它们把同一个输入 X，投影到三个不同的「语义子空间」，分别扮演查询、键、值三种角色。

01 为什么必须是三个不同的矩阵

新手最容易问的问题：既然 Q、K、V 都来自 X，为什么不直接用 X 算注意力，非要乘三个矩阵？或者退一步，让 Wq = Wk = Wv 行不行？

答案是：同一个 token，在「提问」「被匹配」「提供信息」时需要的视角是不同的。用一个生活类比：

💡 类比：一个人的三重身份

想象你在一场学术会议上。你想找做某个方向的人（这是 Query 视角），你的名片上印着你的研究方向（这是 Key 视角，让别人能匹配到你），你真正能讲的具体知识是你的论文（这是 Value，别人匹配上你后真正想拿走的东西）。

这三个视角用的「关键词」完全不同。你想找的可能用「应用」描述，名片用「领域」描述，论文用「方法」描述。投影矩阵就是让模型为同一个 token 学出这三套不同的描述方式。

如果 Wq = Wk，那么 Q = K，注意力分数矩阵就变成 Q·Q^T——一个对称矩阵，每个 token 对对方的关注度和对方对它的关注度完全一样。这在语义上往往是错的：主语关注谓语，和谓语关注主语，强度不该一样。

如果 Wv 也一样，那么 V = X，输出就是输入的加权平均，表达力大幅下降——模型失去了「把信息变换成便于聚合的形式」的能力。

数学视角：解耦三个语义子空间

从线性代数看，每个 W 矩阵是一次投影 + 降维（通常 d_k < d_model）。它把 X 投影到一个 d_k 维的子空间，而这个子空间由训练学到的 W 决定。

三个 W 互不相同，意味着：

  • Wq 学出了一个「查询子空间」：提取 token 里适合「去匹配别人」的特征；
  • Wk 学出了一个「键子空间」：提取 token 里适合「被别人匹配」的特征；
  • Wv 学出了一个「值子空间」：提取 token 里适合「被聚合传播」的特征。

这三个子空间各司其职，让模型拥有了「用一种特征去查询、用另一种特征被匹配、再用第三种特征传递信息」的灵活性。这是单矩阵做不到的。

02 三种角色的具体职责

把 Q、K、V 放进真实的语言任务里，它们的分工会更清晰。

角色投影矩阵学到什么语言例子
Query（查询）Wq「我在找什么样的上下文」代词「它」找指代的名词
Key（键）Wk「我能被什么样的查询匹配到」名词「猫」暴露自己的特征给代词
Value（值）Wv「匹配上后，我传递什么信息」名词把「性别、单复数」语义传给代词

举个指代消解的例子。句子「猫追老鼠，它很饿」。当处理「它」时：

  • 「它」的 Q（经过 Wq 投影）会编码「我在找一个单数、有生命、能饿的名词」这样的查询特征；
  • 「猫」的 K（经过 Wk 投影）会暴露「我是单数、动物、主语」这样的被匹配特征；
  • Q·K 点积大，于是注意力把权重集中在「猫」上；
  • 「猫」的 V（经过 Wv 投影）携带的语义（动物、单数）流入「它」的表示，后续就能正确推断「它=猫」。

注意Wq 学的是「找」，Wk 学的是「被找」，Wv 学的是「传什么」——三件事，三种特征，必须三套矩阵。这就是为什么三个投影不可合并。

03 维度的选择：d_model、d_k、d_v

这里有几个容易混淆的维度，讲清楚：

  • d_model：模型的整体隐藏维度，也就是 X 的列数。GPT-3 是 12288，小模型可能是 512、768。
  • d_k：Q 和 K 投影后的维度，决定点积的维度。实践常取 d_model / num_heads（多头时每头分一份）。
  • d_v：V 投影后的维度，决定输出的维度。通常设 d_v = d_k 简化实现，但理论上可以不同。

原始 Transformer 论文里，d_model=512，使用 8 头注意力，每头的 d_k = d_v = 512/8 = 64。8 头拼回来还是 512 维，再过一个输出投影 W_o 回到 d_model。

📌 d_k 为什么小于 d_model

两个原因。第一，控制计算量：Q·K^T 的复杂度是 O(n²·d_k)，d_k 越小越快。第二，多头并行：把 d_model 拆成 num_heads 个 d_k，每头独立做注意力，再拼回来——这就是多头注意力（下一章的专题）。d_k 小，每头的「视角」更聚焦，多头才有意义。

04 还有第四个矩阵：输出投影 Wo

注意力算完，输出形状是 (n, d_v)。如果 d_v = d_model，可以直接用；但多头场景下，每头输出是 d_v，拼起来是 num_heads × d_v = d_model。为了让输出能加进残差连接（X + Attention(X)），还需要一个输出投影矩阵 Wo，把拼接后的向量再投回 d_model：

output = Concat(head_1, ..., head_h) · Wo

Wo 是 (d_model × d_model) 的矩阵。所以严格说，一个注意力层有四个可学习矩阵：Wq、Wk、Wv、Wo。其中前三者把输入「一拆为三」，Wo 把多头结果「合而为一」。

05 代码实现：从零写一个投影层

把上面所有概念落到代码上。这是一个单头注意力层的完整实现，包含三个投影矩阵：

attention_with_projection.py —— 带投影矩阵的注意力
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class SingleHeadAttention(nn.Module):
    def __init__(self, d_model, d_k=None, d_v=None):
        super().__init__()
        d_k = d_k or d_model
        d_v = d_v or d_model
        # 三个独立的投影矩阵 —— 参数各不相同
        self.W_q = nn.Linear(d_model, d_k, bias=False)
        self.W_k = nn.Linear(d_model, d_k, bias=False)
        self.W_v = nn.Linear(d_model, d_v, bias=False)
        # 输出投影（d_v 回到 d_model）
        self.W_o = nn.Linear(d_v, d_model, bias=False)

    def forward(self, x, mask=None):
        # x: (batch, seq, d_model)
        Q = self.W_q(x)   # (batch, seq, d_k)
        K = self.W_k(x)   # (batch, seq, d_k)
        V = self.W_v(x)   # (batch, seq, d_v)

        d_k = Q.size(-1)
        scores = Q @ K.transpose(-2, -1) / math.sqrt(d_k)  # (batch, seq, seq)
        if mask is not None:
            scores = scores.masked_fill(mask == 0, float('-inf'))
        weights = F.softmax(scores, dim=-1)
        out = weights @ V  # (batch, seq, d_v)
        return self.W_o(out), weights

# 验证：三个矩阵确实不同
torch.manual_seed(42)
layer = SingleHeadAttention(d_model=8, d_k=4, d_v=4)
print("Wq 参数数:", layer.W_q.weight.numel())   # 4*8 = 32
print("Wk 参数数:", layer.W_k.weight.numel())   # 32
print("Wv 参数数:", layer.W_v.weight.numel())   # 32
print("Wo 参数数:", layer.W_o.weight.numel())   # 8*4 = 32

x = torch.randn(2, 5, 8)  # batch=2, seq=5, d_model=8
out, w = layer(x)
print("输出形状:", out.shape)  # (2, 5, 8) —— 和输入一样，方便残差连接

注意三个关键点：①三个 nn.Linear 是独立的、参数不共享的；②输入是 x 一个东西，输出变成 Q、K、V 三个；③最后 W_o 把输出投回 d_model，保证可以做残差。

06 实验：观察三个投影学到了不同的东西

口说无凭，我们做个小实验验证「三个投影矩阵确实不同」。用一个简单任务训练上面的注意力层，然后看 Wq、Wk、Wv 学完后的余弦相似度：

compare_projections.py —— 检验三套矩阵学得有多不同
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

# 一个极简的「复制任务」：让注意力层学会把第一个 token 的信息传到所有位置
torch.manual_seed(0)
d_model = 16
attn = SingleHeadAttention(d_model)  # 复用上一段定义的类

opt = torch.optim.Adam(attn.parameters(), lr=1e-2)
for step in range(300):
    x = torch.randn(8, 6, d_model)            # batch=8, seq=6
    target = x[:, 0:1, :].expand_as(x)        # 目标：所有位置都等于第一个 token
    out, _ = attn(x)
    loss = F.mse_loss(out, target)
    opt.zero_grad(); loss.backward(); opt.step()

# 训练完后，看三个投影矩阵有多「不一样」
def cos_sim(a, b):
    a, b = a.flatten(), b.flatten()
    return F.cosine_similarity(a, b, dim=0).item()

print(f"训练损失: {loss.item():.4f}")
print(f"Wq vs Wk 相似度: {cos_sim(attn.W_q.weight, attn.W_k.weight):.3f}")
print(f"Wq vs Wv 相似度: {cos_sim(attn.W_q.weight, attn.W_v.weight):.3f}")
print(f"Wk vs Wv 相似度: {cos_sim(attn.W_k.weight, attn.W_v.weight):.3f}")

你会发现训练后三者的相似度都很低（通常绝对值小于 0.3，甚至接近 0）。这说明模型主动把三个矩阵推到了不同的方向，因为同一个矩阵表达不了「查询」「被匹配」「传值」三件事。

07 什么时候可以共享投影

虽然标准 Transformer 用三个不同的矩阵，但某些变体会故意共享部分投影，以省参数或加速推理：

变体共享方式目的
Linear Attention无 softmax，Q/K 用核函数O(n²) → O(n)，降复杂度
MQA（第 99 章）所有头共享同一组 Wk、Wv省 KV cache 显存，推理加速
GQA（第 99 章）若干头共享一组 Wk、WvMQA 和 MHA 的折中
Self-Attention 共享 QKWq = Wk（少见）省参数但表达力下降

这些「共享」都是性能与效果的权衡。标准 GPT 类模型默认用三个独立矩阵，只有在显存/速度压力大时才考虑共享——这背后是 LLM 推理优化的一个大话题，我们在第 99 章（MQA/GQA）和第 100 章（KV Cache）会深入展开。

⚠️ Cross-Attention 里 Q 和 K、V 来源不同

本章讲的是 Self-Attention（自注意力），Q、K、V 都来自同一个 X。但在Cross-Attention（交叉注意力）里（比如 Decoder 关注 Encoder 的输出、或图文混合模型里文本关注图像），Q 来自一个序列 X1，而 K、V 来自另一个序列 X2：Q = X1·Wq，K = X2·Wk，V = X2·Wv。投影矩阵仍然各司其职，只是输入源不同。这是翻译、多模态模型的基础。

08 投影层到底用了多少参数

把参数数算清楚，对理解模型规模有帮助。一个标准的单头注意力层：

参数数 = 4 × d_model × d_model
（Wq: d_model×d_k，Wk: d_model×d_k，Wv: d_model×d_v，Wo: d_v×d_model；当 d_k = d_v = d_model 时就是 4d_model²）

以 GPT-3 的 d_model=12288 为例（简化为单头算）：4 × 12288² ≈ 6 亿参数每一层。GPT-3 有 96 层，光注意力投影就是 600 亿参数级别——这解释了为什么大模型动辄上百亿、上千亿参数。投影矩阵是大头之一（另一大头是 FFN，第 102 章讲）。

这也是为什么MQA/GQA 想方设法共享 Wk、Wv：这两个矩阵每个头一份，头数多时累积可观。共享后参数和显存都能显著下降。

★ QKV 投影矩阵初始化

Q/K/V 三个投影矩阵的初始化方式直接影响训练能否起步。下面这个模块对比 Xavier、Kaiming、小尺度正态三种初始化，并解释为什么 Transformer 普遍用 std=0.02：

import torch
import torch.nn as nn
import math

class QKVProjection(nn.Module):
    """把 hidden 维输入投影成 Q/K/V，演示三种初始化的差异。"""
    def __init__(self, hidden: int, n_heads: int, head_dim: int):
        super().__init__()
        self.n_heads = n_heads
        self.head_dim = head_dim
        proj_out = n_heads * head_dim
        # 三套独立权重（实际里 Q/K/V 也可合并成一个大矩阵分块）
        self.W_q = nn.Linear(hidden, proj_out, bias=False)
        self.W_k = nn.Linear(hidden, proj_out, bias=False)
        self.W_v = nn.Linear(hidden, proj_out, bias_out=False) \
                   if False else nn.Linear(hidden, proj_out, bias=False)

    def init_xavier(self):
        """Xavier/Glorot：方差 = 2/(fan_in+fan_out)，适合 tanh/sigmoid。"""
        for m in [self.W_q, self.W_k, self.W_v]:
            nn.init.xavier_uniform_(m.weight)

    def init_kaiming(self):
        """Kaiming/He：方差 = 2/fan_in，配合 ReLU，深层不消失。"""
        for m in [self.W_q, self.W_k, self.W_v]:
            nn.init.kaiming_uniform_(m.weight, a=math.sqrt(5))

    def init_scaled_normal(self, scale: float = 0.02):
        """小尺度正态：GPT/BERT 系默认 std=0.02，保证初始注意力接近均匀。"""
        for m in [self.W_q, self.W_k, self.W_v]:
            nn.init.normal_(m.weight, mean=0.0, std=scale)

    def forward(self, x):
        B, S, _ = x.shape
        q = self.W_q(x).view(B, S, self.n_heads, self.head_dim).transpose(1, 2)
        k = self.W_k(x).view(B, S, self.n_heads, self.head_dim).transpose(1, 2)
        v = self.W_v(x).view(B, S, self.n_heads, self.head_dim).transpose(1, 2)
        return q, k, v

# 对比三种初始化下投影输出的方差
proj = QKVProjection(hidden=4096, n_heads=32, head_dim=128)
x = torch.randn(2, 16, 4096)

for name, fn in [("Xavier", proj.init_xavier),
                 ("Kaiming", proj.init_kaiming),
                 ("Normal0.02", proj.init_scaled_normal)]:
    fn()
    q, k, v = proj(x)
    print(f"{name:11s} Q.std={q.std():.4f} K.std={k.std():.4f} V.std={v.std():.4f}")
# Transformer 通常用 Normal(0,0.02)：初始时 Q·K^T/√d ≈ 小数，
# softmax 近似均匀，梯度健康，是 GPT/Llama 系的共识选择

09 本章小结

这一章的核心结论：

三个矩阵的本质
  • Wq：投影出「查询」特征，让 token 知道自己想找什么
  • Wk：投影出「键」特征，让 token 能被别人匹配
  • Wv：投影出「值」特征，让 token 知道匹配后传什么
  • 三者参数不共享，各学各的语义子空间
为什么要三个不同
  • 同一个 token 在「提问」「被匹配」「传值」时视角不同
  • 共享会让分数矩阵对称，丧失非对称关注能力
  • V 共享会丢失「变换信息便于聚合」的能力
  • MQA/GQA 在推理优化时才刻意共享，是效果换速度

到这里，QKV 投影的全貌就讲完了。但你可能注意到一个大问题：QK^T 这个点积，对 token 的顺序完全无感——把「我打你」和「你打我」输入进去，如果不告诉模型谁在前谁在后，点积算出的注意力分数是一样的。也就是说，Transformer 本身是不知道位置信息的。这是个要命的缺陷。下一章我们讲位置编码——怎么把「第几个 token」这层信息重新塞回去。
