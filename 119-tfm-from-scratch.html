从零实现迷你 Transformer
用 300 行代码拼出一个能写诗的语言模型

读了十八章的理论，是时候把所有零件亲手焊在一起了。本章不调任何框架的高级 API，只用 PyTorch 的基础算子，从 Embedding、位置编码、多头注意力、FFN、LayerNorm 一路实现到一个完整的 Decoder-Only 语言模型，并配上训练循环和生成函数。最后我们让它在 CPU 上学一段莎士比亚，亲眼看它吐出像模像样的诗句。这是理解前面所有章节的最佳实践。

★ 一、我们要造什么

目标很明确：从零实现一个 Decoder-Only 的迷你 GPT。它包含一个完整的语言模型该有的所有部件：词表和 Embedding、位置编码、若干层 Transformer Block（每层含多头因果注意力 + FFN + LayerNorm）、最终的输出投影。配上训练循环（前向 + 损失 + 反向）和生成函数（greedy 和采样），它就能在一个小语料上训练并生成文本。

我们的设计原则有三条。第一，不依赖任何高级封装——不用 nn.Transformer，不用 Hugging Face，所有部件手写，让你看清每个细节。第二，可在 CPU 上跑通——模型尺寸控制得很小，普通笔记本几分钟就能训练，无需 GPU。第三，代码即教学——每个组件对应前面章节讲过的概念，读完代码就等于复习了一遍整个系列。

1.1 模型配置一览

我们的迷你 GPT 配置如下：词表大小 256（用字节级 tokenization，省去分词器复杂度），模型维度 d_model=64，4 个注意力头，2 层 Transformer Block，FFN 隐藏维度 256，最大序列长度 128。这个配置小到能在 CPU 上秒级训练，又大到能展示 Transformer 的所有核心机制。整个模型参数量约几十万，训练几分钟后就能看到 loss 下降、生成文本变得像样。

★ 二、Embedding 与位置编码

Transformer 的输入是 token id 序列，第一步要把它转成连续向量。这需要两个 embedding 相加：词 embedding（把每个 token id 映射成一个 d_model 维向量）和位置 embedding（告诉模型每个 token 处在序列的哪个位置）。

2.1 为什么需要位置编码

注意力机制本身是「位置无关」的——它只看内容相似度，不知道谁在前谁在后。但语言的顺序至关重要（「狗咬人」和「人咬狗」截然不同），所以必须显式注入位置信息。我们这里采用最经典的做法：可学习的位置 embedding。每个位置 0,1,2,...,max_len 对应一个可训练的向量，加到词 embedding 上。

2.2 实现 Embedding 层

import torch
import torch.nn as nn

class TokenPositionEmbedding(nn.Module):
    """词 embedding + 可学习位置 embedding。"""
    def __init__(self, vocab_size, d_model, max_len):
        super().__init__()
        self.token_emb = nn.Embedding(vocab_size, d_model)
        self.pos_emb = nn.Embedding(max_len, d_model)
        self.max_len = max_len

    def forward(self, token_ids):
        """
        token_ids: (batch, seq_len)
        返回: (batch, seq_len, d_model)
        """
        B, S = token_ids.shape
        positions = torch.arange(S, device=token_ids.device).unsqueeze(0)  # (1, seq_len)
        return self.token_emb(token_ids) + self.pos_emb(positions)

# 测试
emb = TokenPositionEmbedding(vocab_size=256, d_model=64, max_len=128)
x = torch.randint(0, 256, (2, 10))
print("Embedding 输出形状:", emb(x).shape)   # (2, 10, 64)

这段代码极其简单，但它完成了 Transformer 的入口。一个细节：位置 embedding 的长度 max_len 要大于实际训练时用到的最大序列长度，否则超出范围会报错。这也是为什么大模型有「最大上下文长度」的限制——超过就没位置向量可用了。

★ 三、多头因果注意力

注意力是 Transformer 的心脏。我们要实现一个完整的、带因果掩码的多头注意力。这一部分对应前面第 13 章讲的内容，这里是它的完整落地。

3.1 多头注意力的完整实现

import torch.nn.functional as F
import math

class MultiHeadCausalAttention(nn.Module):
    def __init__(self, d_model, num_heads):
        super().__init__()
        assert d_model % num_heads == 0
        self.d_model = d_model
        self.num_heads = num_heads
        self.d_head = d_model // num_heads
        self.qkv = nn.Linear(d_model, 3 * d_model)
        self.out = nn.Linear(d_model, d_model)
        # 预计算因果掩码缓存，避免每次 forward 都算
        self.register_buffer("causal", None, persistent=False)

    def _get_causal_mask(self, seq_len, device):
        if self.causal is None or self.causal.shape[0] < seq_len:
            mask = torch.full((seq_len, seq_len), float("-inf"), device=device)
            mask = torch.triu(mask, diagonal=1)
            self.causal = mask
        return self.causal[:seq_len, :seq_len]

    def forward(self, x):
        """
        x: (batch, seq_len, d_model)
        返回: (batch, seq_len, d_model)
        """
        B, S, D = x.shape
        qkv = self.qkv(x).reshape(B, S, 3, self.num_heads, self.d_head)
        qkv = qkv.permute(2, 0, 3, 1, 4)   # (3, B, heads, S, head_dim)
        q, k, v = qkv[0], qkv[1], qkv[2]
        # 缩放点积注意力
        scores = (q @ k.transpose(-2, -1)) / math.sqrt(self.d_head)
        mask = self._get_causal_mask(S, x.device)
        scores = scores + mask.unsqueeze(0).unsqueeze(0)   # 广播到 (1,1,S,S)
        attn = F.softmax(scores, dim=-1)
        out = attn @ v                           # (B, heads, S, head_dim)
        out = out.transpose(1, 2).reshape(B, S, D)
        return self.out(out)

# 测试
attn = MultiHeadCausalAttention(d_model=64, num_heads=4)
x = torch.randn(2, 10, 64)
print("注意力输出形状:", attn(x).shape)   # (2, 10, 64)

这里有几个工程细节值得注意。第一，QKV 投影合并成一个矩阵乘（3*d_model），比分三次投影更高效。第二，因果掩码做了缓存，避免每次 forward 重复构造。第三，掩码通过 unsqueeze 广播到 heads 维度，保证每个头共用同一个掩码。这些细节都是工业级实现的做法。

★ 四、FFN 与 LayerNorm

有了注意力，还需要 FFN（前馈网络）和 LayerNorm 才能凑成一个完整的 Transformer Block。这两个组件相对简单，但各有讲究。

4.1 FFN：两层的逐位置 MLP

FFN 是一个两层的全连接网络，中间有激活函数（通常用 GELU 或 ReLU）。它对每个位置独立作用（不跨 token 交互），起到「逐 token 的非线性变换」作用。FFN 的隐藏维度通常是 d_model 的 4 倍，这是 Transformer 论文里的经验值。

class FeedForward(nn.Module):
    def __init__(self, d_model, d_ff):
        super().__init__()
        self.fc1 = nn.Linear(d_model, d_ff)
        self.fc2 = nn.Linear(d_ff, d_model)

    def forward(self, x):
        return self.fc2(F.gelu(self.fc1(x)))

ffn = FeedForward(d_model=64, d_ff=256)
x = torch.randn(2, 10, 64)
print("FFN 输出形状:", ffn(x).shape)

4.2 LayerNorm 与 Pre-Norm 结构

LayerNorm 对每个位置的向量做归一化（减均值、除标准差、再缩放平移），稳定训练。Transformer 有两种主流结构：Post-Norm（原论文，残差后再 LayerNorm）和 Pre-Norm（现代大模型普遍采用，先 LayerNorm 再做注意力/FFN，最后残差）。我们采用 Pre-Norm，因为它训练更稳定，是 GPT、LLaMA 等现代模型的标准做法。

class TransformerBlock(nn.Module):
    """Pre-Norm 的 Transformer Block：LN -> Attn -> 残差 -> LN -> FFN -> 残差。"""
    def __init__(self, d_model, num_heads, d_ff):
        super().__init__()
        self.norm1 = nn.LayerNorm(d_model)
        self.attn = MultiHeadCausalAttention(d_model, num_heads)
        self.norm2 = nn.LayerNorm(d_model)
        self.ffn = FeedForward(d_model, d_ff)

    def forward(self, x):
        x = x + self.attn(self.norm1(x))      # 残差 + 注意力
        x = x + self.ffn(self.norm2(x))       # 残差 + FFN
        return x

block = TransformerBlock(d_model=64, num_heads=4, d_ff=256)
x = torch.randn(2, 10, 64)
print("Block 输出形状:", block(x).shape)

注意残差连接（x + ...）至关重要——它让梯度能直通到底层，避免深层网络的梯度消失。没有残差，Transformer 训不深。Pre-Norm 的好处是：残差主干上的数据始终是未归一化的原始尺度，信号传递更顺畅。

★ 五、组装完整的迷你 GPT

把所有部件拼起来，就得到一个完整的语言模型。它接收 token id 序列，输出每个位置预测下一个 token 的 logits。

class MiniGPT(nn.Module):
    def __init__(self, vocab_size=256, d_model=64, num_heads=4,
                 num_layers=2, d_ff=256, max_len=128):
        super().__init__()
        self.emb = TokenPositionEmbedding(vocab_size, d_model, max_len)
        self.blocks = nn.ModuleList([
            TransformerBlock(d_model, num_heads, d_ff)
            for _ in range(num_layers)
        ])
        self.norm_final = nn.LayerNorm(d_model)
        # 输出投影：权重与 token embedding 共享（weight tying），提升效果省参数
        self.lm_head = nn.Linear(d_model, vocab_size, bias=False)
        self.lm_head.weight = self.emb.token_emb.weight
        self.max_len = max_len

    def forward(self, token_ids, targets=None):
        """
        token_ids: (batch, seq_len)
        targets: (batch, seq_len) 或 None
        返回: logits (batch, seq_len, vocab_size)；若有 targets 还返回 loss。
        """
        x = self.emb(token_ids)
        for block in self.blocks:
            x = block(x)
        x = self.norm_final(x)
        logits = self.lm_head(x)              # (batch, seq_len, vocab)
        loss = None
        if targets is not None:
            # 把 batch 和 seq_len 维度合并，算交叉熵
            B, S, V = logits.shape
            loss = F.cross_entropy(logits.reshape(-1, V), targets.reshape(-1))
        return logits, loss

# 实例化并打印参数量
model = MiniGPT()
n_params = sum(p.numel() for p in model.parameters())
print(f"模型参数量: {n_params:,}")
# 大约几十万参数，CPU 训练无压力

这里用了一个技巧叫 weight tying（权重共享）：输出投影 lm_head 的权重和输入的 token embedding 共享。这不仅省参数（词表大时省得可观），还能提升效果——因为输入和输出在同一个语义空间里。这是现代大模型（GPT-2 以后）的标配。

★ 六、训练循环：让模型学会续写

有了模型，还需要训练循环和语料。我们用最简单的方式：字节级 tokenization（每个字节是一个 token，词表大小固定 256），语料用一段莎士比亚风格的英文。训练目标是预测下一个 token——这正是 Decoder-Only 模型的核心任务。

6.1 数据准备

# 一段示例语料（实际训练时可换成更长的文本）
text = """
To be, or not to be, that is the question:
Whether 'tis nobler in the mind to suffer
The slings and arrows of outrageous fortune,
Or to take arms against a sea of troubles
And by opposing end them. To die—to sleep,
No more; and by a sleep to say we end
The heart-ache and the thousand natural shocks
That flesh is heir to: 'tis a consummation
Devoutly to be wish'd. To die, to sleep;
To sleep, perchance to dream—ay, there's the rub:
For in that sleep of death what dreams may come,
"""

# 字节级编码：文本 -> 字节序列 -> token id
data = torch.tensor(list(text.encode("utf-8")), dtype=torch.long)
print(f"语料长度: {len(data)} 字节, 词表: 0-255")

def get_batch(data, batch_size, seq_len):
    """随机采样一个 batch：输入 x 和目标 y（x 左移一位）。"""
    ix = torch.randint(0, len(data) - seq_len - 1, (batch_size,))
    x = torch.stack([data[i:i+seq_len] for i in ix])
    y = torch.stack([data[i+1:i+seq_len+1] for i in ix])
    return x, y

x, y = get_batch(data, batch_size=4, seq_len=32)
print(f"batch 输入形状: {x.shape}, 目标形状: {y.shape}")

6.2 训练循环

import torch.optim as optim

model = MiniGPT(vocab_size=256, d_model=64, num_heads=4, num_layers=2, d_ff=256, max_len=128)
optimizer = optim.AdamW(model.parameters(), lr=3e-3)

# 训练 2000 步，每 200 步打印一次 loss
model.train()
for step in range(2000):
    x, y = get_batch(data, batch_size=16, seq_len=64)
    logits, loss = model(x, y)
    optimizer.zero_grad()
    loss.backward()
    # 梯度裁剪，防止训练不稳定
    torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
    optimizer.step()
    if step % 200 == 0:
        print(f"step {step:4d} | loss {loss.item():.4f}")

print("训练完成！")

运行这段代码，你会看到 loss 从约 5.5（初始随机）逐步下降到 1.5 以下。这说明模型学会了语料的统计规律——某些字母组合（如「th」「ing」）出现得更频繁，它学会了预测它们。我们的语料太短，模型没法学会真正的「莎士比亚风格」，但能学会基本的字母模式，这足以验证实现正确。

★ 七、生成函数：让模型开口说话

训练完，我们要让它生成文本。实现两种生成方式：greedy（确定性）和采样（带温度、top-k）。这对应第 110 章讲过的生成策略，这里是它们的从零实现。

@torch.no_grad()
def generate(model, prompt_ids, max_new_tokens=100, temperature=1.0, top_k=None):
    """
    自回归生成。
    prompt_ids: (1, prompt_len) 起始 token 序列
    返回: (1, prompt_len + max_new_tokens) 生成的完整序列
    """
    model.eval()
    ids = prompt_ids
    for _ in range(max_new_tokens):
        # 只取最后 max_len 个 token，避免超出位置 embedding 范围
        cond = ids[:, -model.max_len:]
        logits, _ = model(cond)
        logits = logits[0, -1] / temperature          # 取最后一个位置
        if top_k is not None:
            v, _ = torch.topk(logits, top_k)
            logits[logits < v[-1]] = float("-inf")    # 只保留 top-k
        probs = F.softmax(logits, dim=-1)
        next_id = torch.multinomial(probs, num_samples=1).unsqueeze(0)
        ids = torch.cat([ids, next_id], dim=1)
    return ids

# 用 "To be" 作为 prompt 生成
prompt = torch.tensor([list("To be, or not".encode("utf-8"))], dtype=torch.long)
out_ids = generate(model, prompt, max_new_tokens=80, temperature=0.8, top_k=40)
print("生成结果：")
print(out_ids[0].tolist().__len__(), "个 token")
print(bytes(out_ids[0].tolist()).decode("utf-8", errors="replace"))

因为语料很短，生成结果不会是完美的莎士比亚，但你会看到它生成的字母序列符合英语的字母分布（大量 e、t、a、空格，合理的辅音元音搭配），偶尔蹦出像「the」「to」「and」这样的常见词。这证明模型确实学到了语言的结构。

★ 八、如何验证实现正确

从零实现容易出 bug，而且很多 bug 不会报错，只是让效果变差。下面几个测试能帮你确认实现正确。

8.1 检查因果掩码没有泄漏

一个关键测试：确认因果掩码真的防止了「偷看未来」。方法是构造一个序列，打乱某个未来位置的 token，观察当前位置的输出是否变化——如果掩码正确，当前位置的输出不应受未来 token 影响。

model.eval()
seq = torch.randint(0, 256, (1, 10))
# 取位置 3 的输出
with torch.no_grad():
    logits_orig, _ = model(seq)
    out_pos3_orig = logits_orig[0, 3]

# 修改位置 5 之后的所有 token（未来）
seq_modified = seq.clone()
seq_modified[0, 5:] = torch.randint(0, 256, (5,))
with torch.no_grad():
    logits_mod, _ = model(seq_modified)
    out_pos3_mod = logits_mod[0, 3]

diff = (out_pos3_orig - out_pos3_mod).abs().max()
print(f"修改未来 token 后，位置 3 输出的最大差异: {diff.item():.2e}")
# 如果掩码正确，差异应该是 0（或极小的浮点误差）

8.2 检查 loss 能持续下降

第二个测试更宏观：确认训练 loss 能持续下降到合理水平。如果 loss 卡住不动或者反弹，说明实现有 bug（梯度爆炸、掩码错误、初始化不当等）。我们的训练循环里已经打印了 loss，正常情况下应该从 5.5 平滑下降到 1.5 左右。如果下降不到 3 以下，多半是注意力或 FFN 实现有问题。

8.3 检查梯度流动

第三个测试：确认梯度能从输出层流到输入层。深层网络容易梯度消失，可以打印每层参数的梯度范数，确认它们没有塌缩到 0。

model.train()
x, y = get_batch(data, batch_size=8, seq_len=32)
_, loss = model(x, y)
loss.backward()
print("各层梯度范数：")
for name, p in model.named_parameters():
    if p.grad is not None and p.dim() > 1:   # 只看权重矩阵
        print(f"  {name:40s} grad_norm = {p.grad.norm().item():.4f}")
# 正常情况下，从输出层到 embedding 层，梯度范数应该在同一数量级

★ 九、从这里走向真实大模型

这个迷你 GPT 虽小，但骨架和 GPT-2、LLaMA 完全一致。从它出发，要走向真实的大模型，需要叠加一系列工程优化。这些优化不改变核心架构，但让模型能 scale 到百亿参数。

  • 分词器：用 BPE 或 SentencePiece 把字节级升级到子词级（词表 3 万到 15 万），大幅提高压缩率和效果。
  • 旋转位置编码（RoPE）：替代可学习位置 embedding，支持外推到更长序列，是 LLaMA 等模型的标准。
  • RMSNorm：替代 LayerNorm，省去均值计算，更快，效果相当。
  • SwiGLU 激活：替代 GELU，在 FFN 里引入门控，效果更好（LLaMA 采用）。
  • GQA / MQA：分组查询注意力，减少 KV cache 显存，加速推理。
  • Flash Attention：分块计算注意力，避免实例化完整 attention 矩阵，省显存、加速。
  • 张量并行 / 流水线并行：把模型切分到多张 GPU，训练超大模型。

每一个优化都对应一篇论文和一个工程突破。但它们的根基，都是你今天亲手实现的这个迷你 GPT。理解了它，再去看 LLaMA、DeepSeek 的源码，你会发现核心逻辑你已经会了，只是多了这些优化层。
