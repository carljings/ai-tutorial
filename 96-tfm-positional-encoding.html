位置编码
没有它 Transformer 就是个瞎子

上一章我们留下一个致命问题：注意力的点积对 token 顺序完全无感，「我打你」和「你打我」算出来的注意力分数一模一样。这意味着 Transformer 天生是个「语序瞎子」。本章讲清楚为什么位置编码不可或缺、原论文的正弦余弦编码是怎么设计的、为什么相邻维度用不同频率、以及它的致命弱点——外推性差，这直接催生了后来的 RoPE 和 ALiBi。

阅读约 27 分钟 难度 ★★★★ 前置：第 94、95 章

★ Transformer 为什么是「语序瞎子」

先做个思想实验。把「狗咬人」和「人咬狗」的三个 token embedding 喂给一个不带位置编码的注意力层。会发生什么？

注意力算的是 Q·K^T。注意，矩阵乘法只看行内容，不看行的顺序。把输入矩阵的行打乱重排，得到的分数矩阵只是相应地重排了行列，对应的注意力模式完全一样。也就是说，模型根本分不清哪个 token 在前、哪个在后。

输入 A：[狗, 咬, 人]
▼ 注意力（无位置编码）
注意力分数 = [[a,b,c],[d,e,f],[g,h,i]]
▼ 把行换成 [人, 咬, 狗]
输入 B：[人, 咬, 狗]
▼
分数 = [[c,b,a],[f,e,d],[i,h,g]] —— 只是行列重排，语义结构相同
图 96-1 · 没有位置信息，「狗咬人」和「人咬狗」对模型来说是「同一件事」

这显然不行。中文靠语序区分主客体，英文靠语序区分主谓宾，顺序就是语法，顺序就是语义。RNN 天然按时间步处理，自带顺序；CNN 靠卷积核的局部位置隐含顺序。唯独 Attention 是个「集合操作」——它把输入当成一个无序的词袋。

所以必须人为把位置信息显式塞回输入。这就是位置编码（Positional Encoding，PE）的使命。

位置编码的本质：给序列里每个位置一个独一无二的「身份证号」，并把这个身份证编码进 token 的向量里，让模型能区分「第 3 个位置的猫」和「第 7 个位置的猫」。

01 为什么不直接用整数 1、2、3

最朴素的想法：第 1 个 token 加 1，第 2 个加 2，第 n 个加 n。简单粗暴。但有几个问题：

  • 数值无界：序列一长，位置 1000、10000 的数值远大于 embedding（通常 ±1 量级），会把 embedding 完全淹没。
  • 无法泛化：训练时只见过 0~512，推理时来个 600，模型没见过这个数，行为不可控。
  • 无相对关系：整数 5 和 6 的差是 1，5 和 100 的差是 95——模型很难从绝对数值里学到「相对距离」这种对语言更重要的信息。

那归一化到 [0,1] 呢？又带来新问题：不同长度序列里「第 3 个位置」归一化后是 3/n，n 不同这个值就不同，同一个位置的编码变得依赖于序列总长度，混乱。

位置编码要同时满足几个诉求：①有界（不淹没 embedding）；②每个位置唯一；③能反映相对距离；④最好能外推到训练时没见过的长度。原论文的正弦余弦编码，正是为了同时满足这些而设计的精巧方案。

02 正弦余弦编码：原论文的方案

2017 年《Attention is All You Need》给出了这样的位置编码。对于位置 pos 和维度 i：

PE(pos, 2i)   = sin( pos / 10000^(2i/d_model) )
PE(pos, 2i+1) = cos( pos / 10000^(2i/d_model) )

也就是说，每个位置的编码是一个 d_model 维向量，偶数维用 sin、奇数维用 cos，且不同维度用不同频率。频率由 10000^(2i/d_model) 决定——维度越靠后，频率越低（波长越长）。

直观理解：把这个 d_model 维向量想象成一组指针时钟。前面的维度是「秒针」（高频，转得快，每个位置都不一样），后面的维度是「分针」「时针」（低频，转得慢，几十个位置才走一格）。每个位置都由这组时钟的读数唯一确定。

维度 0,1
高频（秒针）
维度 2,3
较高频
维度 ...
维度 d-2,d-1
低频（时针）
▼ 多个频率组合
每个位置的编码向量唯一，且蕴含相对距离信息
图 96-2 · 多频率编码：像一组不同转速的指针，组合出唯一位置

为什么用多频率

单频率（比如全用 sin(pos/100)）只能在一个尺度上区分位置。但语言里「相对距离」是多尺度的：相邻 token 差 1，句子首尾可能差 100。要同时表达「很近」和「很远」，就需要高频分辨近距离、低频分辨远距离。这就是多频率的用意——和计算机里的浮点数用不同指数位表示大小是类似的思想。

03 为什么 sin/cos 能表达相对位置

正弦余弦编码最巧妙的地方：它隐含了相对位置信息。利用三角恒等式：

sin(pos + k) = sin(pos)cos(k) + cos(pos)sin(k)
cos(pos + k) = cos(pos)cos(k) − sin(pos)sin(k)

写成矩阵形式，PE(pos+k) 可以用 PE(pos) 经过一个只依赖于偏移 k（不依赖 pos）的线性变换得到：

[ sin(pos+k) ]   = [ cos(k)  sin(k)] [ sin(pos) ]
[ cos(pos+k) ]    [−sin(k)  cos(k)] [ cos(pos) ]

这意味着「偏移 k」这个相对信息，被编码进了 sin/cos 之间的稳定关系里。无论从哪个绝对位置 pos 出发，平移 k 步所经历的变换是一样的。模型理论上可以从这种稳定的平移结构里，学出「相对距离」的概念——而相对距离对语言（比如「修饰语离中心词多远」）比绝对位置重要得多。

💡 类比：二进制表示

另一种直观理解。把位置用二进制写出来：000、001、010、011、100……低位翻转快（像高频），高位翻转慢（像低频）。每个数唯一，而且低位承载「细粒度」、高位承载「粗粒度」。sin/cos 编码本质上是二进制的连续可微版本——用正弦波代替方波，让它能通过梯度下降学习。

04 位置编码怎么塞进模型

原论文的做法极其简单粗暴：直接把位置编码向量加到 token embedding 上。

输入 = TokenEmbedding(token) + PositionalEncoding(pos)

是「相加」不是「拼接」。为什么相加也行？因为 d_model 维空间是高维的，token 语义和位置信息可以在这个高维空间里近似正交地共存——加在一起后，模型通过后续的线性变换能把它们「解混」。实验证明相加的效果不比拼接差，且省一半维度。

另外注意，PE 是固定不学习的（原论文版本）。它是用上面公式预先算好的常数矩阵，每个位置的向量在训练中不变。后来也有「可学习位置编码」（Learned PE），把 PE 当成可训练参数——BERT、GPT 早期都用过。可学习 PE 效果略好，但外推性更差（训练时没见过的位置没参数）。

这是 PyTorch 经典实现，可以直接用：

positional_encoding.py —— 正弦余弦位置编码
import torch
import torch.nn as nn
import math

class SinusoidalPositionalEncoding(nn.Module):
    def __init__(self, d_model, max_len=5000):
        super().__init__()
        pe = torch.zeros(max_len, d_model)            # (max_len, d_model)
        position = torch.arange(0, max_len).unsqueeze(1).float()  # (max_len, 1)
        # 10000^(2i/d_model) 的倒数，即频率
        div_term = torch.exp(
            torch.arange(0, d_model, 2).float() * (-math.log(10000.0) / d_model)
        )
        pe[:, 0::2] = torch.sin(position * div_term)  # 偶数维 sin
        pe[:, 1::2] = torch.cos(position * div_term)  # 奇数维 cos
        pe = pe.unsqueeze(0)                           # (1, max_len, d_model)
        self.register_buffer('pe', pe)                 # 不参与训练

    def forward(self, x):
        # x: (batch, seq_len, d_model)
        return x + self.pe[:, :x.size(1)]

# 看看不同位置的编码长什么样
pe = SinusoidalPositionalEncoding(d_model=8, max_len=10)
print("位置 0 的编码:", pe.pe[0, 0])
print("位置 1 的编码:", pe.pe[0, 1])
print("位置 0 和 1 的差异:", (pe.pe[0, 1] - pe.pe[0, 0]))

注意几个细节：①用 register_buffer 而非 Parameter，表示 PE 不参与梯度更新；②用 -log(10000)/d_model 配合 exp 来算频率，等价于 1/10000^(2i/d_model) 但数值更稳定；③偶数位 sin、奇数位 cos 配对。

06 可视化：频率分布与相对距离

我们画两个图直观感受。第一个是不同维度的频率，第二个是位置之间的相似度。

pe_visualize.py —— 可视化位置编码的性质
import torch
import math

d_model = 64
max_len = 100
pe = torch.zeros(max_len, d_model)
position = torch.arange(0, max_len).unsqueeze(1).float()
div_term = torch.exp(torch.arange(0, d_model, 2).float() * (-math.log(10000.0) / d_model))
pe[:, 0::2] = torch.sin(position * div_term)
pe[:, 1::2] = torch.cos(position * div_term)

# 1. 不同维度的「波长」——频率越高，波长越短
for i in [0, 8, 16, 32, 63]:
    freq = 1.0 / (10000 ** (2 * (i // 2) / d_model))
    wavelength = 1.0 / freq if freq > 0 else float('inf')
    print(f"维度 {i:2d}: 周期约 {wavelength:8.1f} 个位置")

# 2. 位置 10 和其他位置的余弦相似度
target = pe[10]
sims = torch.cosine_similarity(target.unsqueeze(0), pe, dim=1)
print("\n位置 10 与各位置的相似度:")
for p in [9, 10, 11, 15, 20, 50, 90]:
    print(f"  pos={p:3d}: {sims[p]:.3f}")

你会看到一个有意思的现象：位置 10 和它附近的位置（9、11、15）相似度高，和远处的位置相似度低且震荡。相似度随距离呈现周期性衰减——这正是多频率编码的特点，近处靠高频分辨，远处靠低频分辨。

07 致命弱点：外推性差

正弦余弦编码理论上能外推——因为公式对任意 pos 都成立，训练时见过 0~512，推理时算 1000 也行。但实际效果很差。原因：

  • 训练分布偏移：模型只学过 0~512 范围内的 PE 模式，超出后 PE 向量的统计特性（数值范围、频率组合）变化剧烈，注意力行为失控。
  • 绝对位置硬编码：相加后，位置信息「焊死」在了绝对位置上。模型学到的注意力模式（比如「关注前一个词」）在长度变化时无法保持。
  • 多层放大：错误在几十层 Transformer 里会指数放大，长序列外推时性能断崖式下降。
⚠️ 这就是为什么现代大模型不用它

正弦余弦编码是 2017 年的开创性方案，但它的外推性缺陷，让它在长上下文时代被淘汰。现代大模型（Llama、GLM、Qwen、Mistral）几乎清一色用 RoPE（旋转位置编码）。RoPE 把位置信息编码进「旋转角度」，让注意力分数自然只依赖相对位置，外推性好得多。这是下一章（第 97 章）的主题。

08 绝对 vs 相对：两条路线

位置编码的发展分成两大流派，理解这个分类对看懂后续章节很关键：

流派代表方法思路特点
绝对位置编码正弦余弦（本章）给每个绝对位置一个向量，加到 embedding简单，但外推差
可学习 PE（BERT/GPT）把 PE 当参数学效果略好，外推更差
相对位置编码相对位置偏置（T5）在注意力分数上加偏置 b[i−j]外推好，但每层都要
RoPE（Llama/GLM）旋转 Q、K 向量分数只依赖相对位置
ALiBi（第 105 章）分数减一个距离惩罚极简，外推极强

趋势很明显：从绝对走向相对。因为语言本质上是相对的——「主语后面跟谓语」「修饰语离中心词 1~2 步」——绝对位置反而不自然。相对位置编码让模型直接学到「隔几个位置」的概念，泛化性和外推性都更好。

09 一个常见疑问：为什么是加不是拼

很多人第一反应是：位置和语义是两件事，应该拼接成 2×d_model 维才合理。原论文选了相加，理由有几条：

  • 省维度：拼接要把 d_model 翻倍，参数和计算都涨。相加不增加维度。
  • 高维空间够大：d_model 通常是几百上千维，语义和位置可以近似正交地「挤」进去，加法不会严重破坏语义。
  • 实验支持：原论文对比过，相加和拼接效果接近，没必要用更贵的拼接。

不过这条经验只适用于早期的相加式 PE。现代相对位置编码（RoPE、ALiBi）已经不再用相加——RoPE 是旋转 Q/K，ALiBi 是改分数，都不动 embedding。这是位置编码思路的进化：从「改输入」到「改注意力计算本身」。

★ 正弦位置编码可视化

正弦位置编码最直观的理解方式是可视化——看不同维度的波形和位置间的相似度。下面用 PyTorch 实现并生成两张图，把"为什么用 sin/cos"和"相对位置怎么体现"讲清楚：

import torch
import matplotlib.pyplot as plt
import numpy as np

def sinusoidal_pe(seq_len: int, d_model: int) -> torch.Tensor:
    """原始 Attention is All You Need 的正弦位置编码。"""
    pe = torch.zeros(seq_len, d_model)
    pos = torch.arange(seq_len).unsqueeze(1).float()
    div = torch.exp(torch.arange(0, d_model, 2).float() *
                    (-np.log(10000.0) / d_model))
    pe[:, 0::2] = torch.sin(pos * div)   # 偶数维 sin
    pe[:, 1::2] = torch.cos(pos * div)   # 奇数维 cos
    return pe

seq_len, d_model = 64, 128
pe = sinusoidal_pe(seq_len, d_model).numpy()

# 图1：每个维度随位置的值（挑 4 个维度看不同频率）
fig, axes = plt.subplots(1, 2, figsize=(13, 4.5))

for d in [0, 4, 16, 64]:
    axes[0].plot(pe[:, d], label=f"dim={d}", lw=1.2)
axes[0].set_xlabel("position")
axes[0].set_ylabel("encoding value")
axes[0].set_title("不同维度的正弦波（低维=高频，高维=低频）")
axes[0].legend(); axes[0].grid(alpha=0.3)

# 图2：位置间点积热力图——验证"相对位置决定相似度"
dot = pe @ pe.T
im = axes[1].imshow(dot, cmap="RdBu_r", vmin=-2, vmax=8, aspect="auto")
axes[1].set_title("位置向量点积（对角线外呈条纹=相对位置模式）")
axes[1].set_xlabel("position j"); axes[1].set_ylabel("position i")
fig.colorbar(im, ax=axes[1], fraction=0.046)

plt.tight_layout()
plt.savefig("sinusoidal_pe.png", dpi=120)
print("已保存 sinusoidal_pe.png")
# 低维频率高 → 编码细粒度位置；高维频率低 → 编码粗粒度位置
# 点积图的条纹说明：PE(i)·PE(j) 只依赖 i-j，这正是相对位置可加性的来源

10 本章小结

这一章讲清了位置编码的来龙去脉：

核心要点
  • 注意力是「集合操作」，天生无序，必须补位置
  • 正弦余弦编码用多频率的 sin/cos 给每个位置唯一向量
  • 三角恒等式让它隐含相对位置信息
  • 做法是直接加到 embedding 上（相加不拼接）
为什么被淘汰
  • 外推性差：超训练长度后效果崩
  • 绝对位置不自然：语言本质是相对的
  • 趋势：绝对 → 相对（RoPE、ALiBi）
  • 现代大模型几乎都用 RoPE

位置编码的故事远没结束。正弦余弦编码留下的「外推性」难题，催生了一整套相对位置编码技术。其中最成功、用得最广的就是 RoPE（Rotary Position Embedding）——Llama、GLM、Qwen 全家桶都在用。它怎么用「旋转」把位置编码进 Q/K，又是怎么优雅地让注意力分数只依赖相对位置？下一章我们从旋转矩阵开始，把 RoPE 彻底拆透。
