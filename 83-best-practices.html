50 条经过血泪验证的黄金法则

全书 82 章的精华，浓缩成 50 条最佳实践——每条都是踩过坑总结出的铁律。按领域分类，每条一句话说清「做什么」和「为什么」。

阅读约 30 分钟难度 ★★★收藏向

01 Prompt 工程（10 条）

#法则原因
1给角色、任务、约束、格式、示例五要素结构化 prompt 远胜散文
2关键指令放 prompt 末尾（近因效应）模型对最后的内容更敏感
3要 JSON 就说清 schema，给示例「返回 JSON」太模糊
4别假设模型知道上下文——给它模型不知道你的业务
5用「不要…」明确边界反向约束比正向引导更有效
6few-shot 选 3-5 个多样化示例太少不够，太多浪费
7prompt 要版本化、测试化它是代码资产，不是一次性
8温度：事实型低（0.1-0.3），创意型高（0.7-1.0）匹配任务性质
9要求模型「不确定就说不确定」减少幻觉
10复杂任务用 CoT（一步步想）分步推理比一步到位更准

02 RAG 工程（10 条）

#法则原因
11分块决定 70% 效果，别随便切语义切分 > 固定长度
12每个 chunk 带元数据（来源/位置）可溯源
13向量+BM25 混合检索，RRF 融合语义+精确双保险
14召回 50→Rerank 精选 5Cross-Encoder 精度高得多
15查询改写（指代消解/HyDE）原始 query 往往不适合直接检索
16用 BGE-m3 等多语言 Embedding中英文都好
17父子分块：小块检索，大块返回精确检索+完整上下文
18回答必须带引用编号可溯源+防幻觉
19用 RAGAS 持续评估不评估就不知道好不好
20增量索引（按 doc_id 更新/删除）不用每次重建全库

03 Agent 工程（10 条）

#法则原因
21简单优先——能用单步别用 Agent复杂度是万恶之源
22必有步数上限+预算上限防失控
23重复检测（连续 3 步相同动作则跳出）防死循环
24工具失败如实喂回，禁止模型臆测防「假装成功」
25用子 Agent 做会产生大量中间过程的子任务上下文隔离
26权限默认拒绝，deny 优先最小权限原则
27高危操作（删除/外发）人工确认不可逆操作不能自动
28指令与外部数据用标签隔离防间接注入
29每步记录 trace（思考/动作/结果）可调试可回放
30PEV：写完就验证，别信第一次可靠性核心

04 成本与性能（10 条）

#法则原因
31模型分级：简单→小模型，难→大模型省 50-80%
32Prompt Caching：固定前缀缓存省 50-90%
33上下文精准裁剪，别全塞越长越贵越差
34流式输出降低感知延迟体感降 80%
35推理用 vLLM（PagedAttention+连续批处理）吞吐 3-10×
36INT4 量化：精度损失 <3%，显存降 4×私有化部署必备
37相同 query 缓存结果（Redis）命中即免费
38意图路由：70% 简单 query 不走 LLM规则/小模型处理
39监控每请求成本，设预算告警防成本失控
40高用量场景私有化比 API 便宜边际成本递减

05 安全与质量（10 条）

#法则原因
41从第一天就建评估集没有评估的优化是盲飞
42每次改动跑评估回归防「修了 A 坏了 B」
43把 bad case 持续加入评估集最宝贵的优化资产
44安全纵深防御：权限+Hook+审计+确认没有银弹，多层叠加
45上线前过红队测试主动找漏洞
46AI 生成内容需标识（合规）法规要求
47数据脱敏（PII）是法律红线隐私保护
48高敏场景（医疗/法律）必须人工审核AI 是辅助非替代
49有回滚方案（模型/prompt/数据）出事能退
50透明告知用户在用 AI信任的基础
