08 clone 后先看什么：核心仓库入口指南

新手最大的痛点不是「找不到仓库」，而是 clone 下来后面对几万行代码不知道从哪看起。下面这份「入口指南」为前面推荐的每个核心仓库，指定一个具体到文件的起点。每个仓库只挑一个最小入口，看完它就足以理解整个项目的骨架。

仓库clone 后先看为什么是这个
karpathy/nanoGPTmodel.py（约 300 行）整个 GPT 架构浓缩在一个文件，从 Embedding 到生成全流程，配合 train.py 一晚就能读完
langchain-ai/langgraphexamples/chat_bot.ipynb用一个聊天机器人把图结构、状态、节点全讲透，比读 SDK 源码快十倍
microsoft/autogenautogen/agentchat/conversable_agent.py多智能体通信的核心抽象全在这个类，generate_reply 是关键方法
chroma-core/chromaexamples/basic_function.py三行代码建库、插入、检索，是理解所有向量库 API 共性的最快路径
FlagOpen/FlagEmbeddingexamples/finetune/embedder/中文 Embedding 微调的官方范例，数据格式和训练脚本可直接复用
modelcontextprotocoldocs/quickstart/server.md30 分钟手写一个 MCP Server，比读规范文档直观
letta-ai/lettaletta/main.py + schemas/memory.py前者是分层记忆的总调度，后者定义了记忆的数据结构，对照看就懂 MemGPT
explodinggradients/ragassrc/ragas/metrics/_faithfulness.py看最核心的「忠实度」指标怎么用 LLM 实现，其它指标套路一样

读源码的一个通用心法：先读 tests/examples，再读 src。tests 和 examples 是作者写给自己的「最小用法」，从它们入手能立刻看清 API 形状，而直接啃 src 容易陷入实现细节。下面这段脚本可以帮你快速定位任意仓库的「最小入口」。

find_entry.py —— 自动定位仓库的最小入口文件
"""
find_entry.py —— clone 一个仓库后，快速找到值得先读的入口文件
思路：优先找 tests/examples/docs 下的最小文件，其次找 src 下带 __main__ 的文件
"""
import os, re
from pathlib import Path

PRIORITY_DIRS = [
    "examples", "example", "docs",
    "tests", "test", "notebooks",
]
SKIP = {"node_modules", ".git", "dist", "build", "__pycache__"}

def score(path):
    """文件越小、目录优先级越高，分数越低（越值得先读）"""
    size = path.stat().st_size
    for i, d in enumerate(PRIORITY_DIRS):
        if f"/{d}/" in str(path):
            return i * 1000 + size
    # src 里有 __main__ 的优先
    text = path.read_text(errors="ignore")
    return (9999 * 1000 + size) - (100000 if "__main__" in text else 0)

def find_entry(repo_dir, top_n=5, max_size=20000):
    files = []
    for root, dirs, fs in os.walk(repo_dir):
        dirs[:] = [d for d in dirs if d not in SKIP]
        for f in fs:
            if not f.endswith((".py", ".ipynb", ".md")):
                continue
            p = Path(root) / f
            if p.stat().st_size <= max_size:
                files.append(p)
    files.sort(key=score)
    return [str(p.relative_to(repo_dir)) for p in files[:top_n]]

09 5 周学习计划：从 API 调用到多 Agent 系统

第 07 节给了五步路径，但只有标题没有可执行细节。这里把它扩成一份带代码任务和验收标准的 5 周计划，每周一个 milestone，做完一个再进下一个。这份计划本身可以用下面的脚本跟踪进度。

five_week_plan.py —— 可跟踪的学习计划
from dataclasses import dataclass
from datetime import date, timedelta

@dataclass
class WeekPlan:
    week: int
    theme: str
    tasks: list
    acceptance: str          # 验收标准：达到什么算过关
    chapters: list           # 对应教程章节

PLAN = [
    WeekPlan(1, "跑通一个模型", [
        "注册智谱开放平台，用 GLM API 完成 1 次对话 + 1 次工具调用",
        "用 tiktoken 数 token，理解上下文窗口",
        "读 nanoGPT 的 model.py，对照本教程第 1 章",
    ], acceptance="能独立写 20 行代码调通 GLM API 并解析 JSON 输出",
       chapters=["01-llm-fundamentals", "02-glm-training"]),

    WeekPlan(2, "手写 Agent Loop", [
        "照第 4 章从零写一个 ReAct Agent，跑通 Plan-Execute-Verify",
        "给它接 2 个工具：search、calc",
        "读 langgraph 的 chat_bot.ipynb 对比官方实现",
    ], acceptance="Agent 能自主决定调哪个工具，并完成 3 步以上的任务",
       chapters=["04-agent-loop", "05-agents"]),

    WeekPlan(3, "搭一个 RAG", [
        "用自己的 10 篇文档搭问答系统",
        "调分块大小、换 BGE Embedding、加 Rerank",
        "跑 RAGAS 评估，记下三维度分数",
    ], acceptance="faithfulness > 0.8，且能定位到至少 1 个可改进维度",
       chapters=["06-rag", "20-vector-database"]),

    WeekPlan(4, "加 Harness 防护", [
        "给 Agent 加权限检查、工具白名单",
        "加错误恢复（重试 + 降级）",
        "跑一次 prompt injection 自测（第 61 章 payload）",
    ], acceptance="能挡住 80% 的注入 payload，且正常请求不被误拒",
       chapters=["03-harness", "61-redteam-handbook"]),

    WeekPlan(5, "造 Skill + 多 Agent", [
        "把高频工作流封装成 1 个 SKILL.md（第 184 章模板）",
        "用 CrewAI 或 LangGraph 搭一个 2-Agent 协作",
        "上线 + 接入反馈收集（第 231 章飞轮）",
    ], acceptance="有一个能跑、能收集 bad case、能迭代的最小闭环系统",
       chapters=["08-skills", "231-br-data-closedloop"]),
]

def print_schedule(start_date=date.today()):
    for i, p in enumerate(PLAN):
        wk_start = start_date + timedelta(weeks=i)
        print(f"\n=== 第 {p.week} 周（{wk_start}）｜{p.theme} ===")
        for t in p.tasks:
            print(f"  - [ ] {t}")
        print(f"  验收：{p.acceptance}")
        print(f"  对应章节：{', '.join(p.chapters)}")

这份计划的关键设计：每周都有可量化的验收标准，而不是「学一下」「了解一下」这种无法判断完成度的目标。这是从「看教程」到「真学会」的最大分水岭——能跑出验收标准里那个数字，才算这一周过关。

⚠ 学习计划的两个纪律

第一，每周只做一个 milestone，不要并行。AI 工程的知识是强依赖的（Agent 依赖 API，RAG 依赖 Agent），并行只会让你每样都半懂。第二，第 5 周的「上线」必须真上线，哪怕是只给自己用的内部服务——只有真用户用了，飞轮才会开始转，你才会遇到真实 bad case，这才是进阶的开始。

★ 一键安装常用 AI 库

把这些库的安装整理成一个脚本，省去逐个查文档的时间。脚本按用途分组，并区分 CPU/GPU，装完还自动验证关键库能否导入：

#!/usr/bin/env bash
# pip 一键安装本教程涉及的常用 AI 库（按用途分组）
# 用法: bash install_ai_libs.sh [cpu|gpu]

MODE="${1:-cpu}"
echo "安装模式: $MODE  (gpu 模式会装 CUDA 版 torch)"

# ---- 1. LLM 推理与调用 ----
pip install -U openai anthropic tiktoken transformers accelerate

# ---- 2. PyTorch (CPU/GPU 分支) ----
if [ "$MODE" = "gpu" ]; then
    pip install torch --index-url https://download.pytorch.org/whl/cu121
else
    pip install torch --index-url https://download.pytorch.org/whl/cpu
fi

# ---- 3. 向量检索与 RAG ----
pip install -U sentence-transformers faiss-cpu chromadb pymilvus lancedb
pip install -U langchain langchain-community llama-index haystack-ai ragas

# ---- 4. 微调与训练 ----
pip install -U peft trl datasets bitsandbytes deepspeed flash-attn --no-build-isolation

# ---- 5. Agent 与工具 ----
pip install -U pydantic tenacity httpx python-dotenv rich

# ---- 6. 多模态 ----
pip install -U pillow opencv-python librosa soundfile

# ---- 7. 实验管理与可视化 ----
pip install -U wandb mlflow tensorboard gradio streamlit

# ---- 8. Python 实用工具 ----
python - <<'PY'
import importlib, sys
libs = ["torch","transformers","faiss","langchain","peft",
        "wandb","gradio","openai","tiktoken","chromadb"]
for m in libs:
    try:
        importlib.import_module(m)
        print(f"  [OK] {m}")
    except Exception as e:
        print(f"  [FAIL] {m}: {e}")
print("\n环境就绪，开始跑教程里的代码吧。")
PY
# 提示：flash-attn 在 CPU 模式装不上属正常，可跳过；bitsandbytes 也仅 Linux/GPU 需要
