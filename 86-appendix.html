08 TOP 20 核心概念速查表

全书 86 章涵盖数百个概念，但真正"高频使用、必须内化"的核心概念大约 20 个。这张速查表把它们浓缩成一句话定义加关键要点，方便你快速回忆和复习。建议打印出来贴在桌前，随时查阅。

TOP 20 核心概念速查表（一句话定义 + 关键点）

【模型与原理】
1. Attention     注意力机制：序列每位置关注所有位置，加权聚合信息。
                  关键：Q/K/V 三矩阵，softmax 归一化，O(n²) 复杂度。
2. Transformer   基于 Attention 的架构，Encoder/Decoder 或 Decoder-only。
                  关键：残差连接 + LayerNorm + FFN，现代 LLM 的骨架。
3. Tokenization  把文本切成模型能处理的离散单元（token）。
                  关键：BPE/SentencePiece，中文一字≈1-2 token。
4. KV Cache      缓存已计算过的 Key/Value，避免重复计算。
                  关键：长上下文的显存大头，PagedAttention 优化。
5. 量化          把 FP16 权重压成 INT8/INT4，省显存加速。
                  关键：GPTQ/AWQ，INT4 是甜点（损失<2%，省4倍）。

【训练】
6. 预训练         海量无标注数据学语言规律（next token prediction）。
                  关键：Scaling Law，数据越多模型越大效果越好。
7. SFT           用指令-回答对教模型"听懂人话、好好回答"。
                  关键：高质量指令数据比量更重要。
8. RLHF/DPO      用人类偏好对齐，让模型回答"讨人喜欢"。
                  关键：RLHF 复杂但强，DPO 简单且接近效果。
9. LoRA          冻结原模型，只训练低秩适配器（省显存微调）。
                  关键：只训 0.1% 参数，效果接近全参微调。

【推理与部署】
10. vLLM         推理引擎，PagedAttention + 连续批处理，2-4× 加速。
                  关键：无损优化，生产部署首选。
11. Speculative Decoding 小模型猜+大模型验，减少串行步骤。
                  关键：小模型要猜得准，否则回退反而更慢。

【Prompt 与 RAG】
12. CoT          Chain of Thought，让模型"一步步想"再答。
                  关键：适合推理任务，"Let's think step by step"。
13. RAG          检索增强生成：先检索相关文档，再喂给 LLM 生成。
                  关键：切块 + 嵌入 + 向量库 + 检索 + 生成。
14. Embedding    把文本转成向量，语义相近的文本向量也相近。
                  关键：检索的基础，选对模型（bge/e5）很关键。
15. Reranker     Cross-encoder 精排，对召回结果重新排序。
                  关键：比向量检索精确，两阶段（召回+精排）标配。

【Agent】
16. Agent Loop   感知→思考→行动→观察的循环，智能体的本质。
                  关键：工具调用是"手脚"，LLM 是"大脑"。
17. ReAct         Reason + Act，边推理边调工具。
                  关键：Thought/Action/Observation 三段式。
18. Reflection   自我反思，生成后批判改进直到达标。
                  关键：Act→Evaluate→Reflect→Retry 循环。
19. Skills       可插拔的指令包，教 agent"怎么做某类任务"。
                  关键：SKILL.md 契约 + 渐进式披露。
20. MCP          模型上下文协议，让 agent 连接外部系统。
                  关键：Skills 是"方法论"，MCP 是"工具箱"。
