08 核心术语的代码速查

纯文字的术语表容易「读完就忘」。这一节把上面最核心的十来个术语变成能跑的代码片段——每段都在 10 行以内，读一遍就能把抽象概念落到具体的张量运算。建议把它们当作「术语的第二个维度」：表格告诉你「是什么」，代码告诉你「怎么算」。

Token 与 BPE：文本如何变成数字

BPE（Byte-Pair Encoding）做的事就是把一段文本切成子词，再映射成整数 id。这是所有大模型输入的起点。

from transformers import AutoTokenizer

tok = AutoTokenizer.from_pretrained("THUDM/glm-4-9b")
ids = tok("多模态大模型")["input_ids"]        # [151331, 36489, 102123, 99489]
print(ids, "→", tok.convert_ids_to_tokens(ids))
# 一个汉字不一定是 1 个 token，中文往往 1.5~2 字/token

Embedding：把整数 id 映射成稠密向量

Embedding 本质就是一张「查表」——用 token id 当行号，从权重矩阵里取出一行。用 numpy 三行就能模拟：

import numpy as np

vocab_size, dim = 32000, 4096
W = np.random.randn(vocab_size, dim).astype(np.float32) * 0.02   # 嵌入矩阵
token_ids = np.array([151331, 36489])
embedding = W[token_ids]           # shape=(2, 4096)，这就是「文本的向量表示」
# 相似词在训练后会在向量空间里靠近，cos(国王)-cos(男人)+cos(女人) ≈ cos(女王)

Self-Attention：Q/K/V 的五行实现

整个 Transformer 的灵魂就是注意力。剥掉所有工程细节，核心公式只有一句：scores = softmax(Q·Kᵀ/√d) · V。

import torch, torch.nn.functional as F

def attention(Q, K, V):                       # Q,K,V: (seq, d)
    d = Q.shape[-1]
    scores = Q @ K.transpose(-1, -2) / (d ** 0.5)  # token 间两两打分
    weights = F.softmax(scores, dim=-1)             # 归一化成注意力权重
    return weights @ V                            # 按权重加权求和 V → 输出

# weights[i][j] 越大，说明 token i 越「关注」token j

RoPE：旋转位置编码的直觉

RoPE（旋转位置编码）通过给 Q、K 乘上一个与位置相关的旋转矩阵，让注意力分数只取决于相对距离，从而天然支持长上下文。它的好处是「外推」——训练时没见过的长序列也能保持稳定。

LayerNorm：稳定训练的归一化

LayerNorm 对每个 token 的特征维度做减均值、除标准差，再缩放。它和 BatchNorm 的区别是：BatchNorm 按批次统计，LayerNorm 按 token 统计，所以 LayerNorm 完全不受 batch size 影响，非常适合自回归生成。

Cross-Entropy：语言模型的损失函数

语言模型的训练目标极简——预测下一个 token。损失就是预测分布和真实 token 的 one-hot 分布的交叉熵：

# logits: 模型对每个候选 token 的打分 (vocab_size,)
# target: 真实的下一个 token id
loss = F.cross_entropy(logits.unsqueeze(0), torch.tensor([target]))
# 等价于 -log(logits[target] 经 softmax 后的概率)
# 训练就是让正确 token 的概率尽量大 → loss 尽量小

LoRA / QLoRA：参数高效微调

LoRA 把要学的权重增量 ΔW 分解成两个小矩阵的乘积 B@A，其中 B∈R^(d×r)、A∈R^(r×k)，秩 r 通常只有 8~64。这样要训练的参数从 d×k 降到 r×(d+k)，缩小几十倍。QLoRA 则在 LoRA 基础上把原始权重量化到 4-bit，让单张消费级显卡也能微调几十亿参数的模型。

RAG：检索增强生成

RAG 的核心是「先检索、再生成」。它解决的是大模型知识过期、幻觉、私有数据不知道的问题。术语表里一连串的 Embedding / Vector DB / HNSW / Rerank / RRF 都是为这条链路服务的：

# RAG 的三步骨架
query_vec = embed(question)                 # 1. 问题转向量
chunks = vector_db.search(query_vec, k=5)   # 2. 向量库召回 top-5 片段
chunks = rerank(question, chunks)           # 3. Cross-Encoder 精排（可选）
answer = llm(question, context=chunks)      # 4. 带上上下文让 LLM 生成

MoE：混合专家的稀疏激活

MoE（Mixture of Experts）让每个 token 只激活少数几个「专家网络」，而不是过整个网络。这样可以在不线性增加计算量的前提下，把模型总参数做得很大。术语表里的「稀疏激活」就是这个意思——比如 8 个专家每次只激活 2 个，FLOPs 只增加一点，但参数容量翻几倍。

Quantization：量化压缩

量化就是把 float16 权重压成 int8 / int4，显存直接省 2~4 倍，速度也会因为访存减少而提升。代价是微小的精度损失——通常 INT8 几乎无损，INT4 在大多数任务上也只掉 1~2 个点。第 16 章有详细的精度对比。

掌握了上面这些「能跑的代码」，再看术语表里的每一行，你都能在脑海里对应出一个具体的张量形状或一次矩阵运算——这才是真正「内化」了一个概念。

09 速查：一图看懂术语间的血缘

把零散的术语串成一张「血缘图」，能帮你建立全局认知。下面这张关系表不是按字母排，而是按因果链排——左边的概念是右边的「前提」或「组成」。顺着读，你能看到 LLM 从底层数学到上层应用的完整演化路径：先有矩阵运算（Attention），再堆成网络（Transformer），喂大量数据（Pretraining），对齐人类偏好（RLHF），接上工具（Agent），最后做成产品（RAG）。

基础概念→ 组合成上层概念→ 服务于
Token + Embedding→文本的向量表示→ 一切 NLP 的输入
Q/K/V + Softmax→Self-Attention→ Transformer 的核心
Attention + FFN + 残差→Transformer 层→ 堆 N 层 = LLM
LLM + 下一 token 预测→Pretraining→ 会「续写」的基座

这张表最大的价值是揭示了一个事实：所有花哨的 AI 应用，根都扎在同一套数学里。当你被层出不穷的新概念（Agent、RAG、MoE、MCP）弄得眼花时，回到这张表，沿着血缘往上回溯，它们最终都会汇流到「矩阵乘法 + 梯度下降」这两条河。这就是为什么本书反复强调要吃透第 1 章——基础牢了，上面的术语表不过是同一套原理的不同包装。
