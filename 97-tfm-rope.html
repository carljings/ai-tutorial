RoPE 旋转位置编码
现代大模型的事实标准

Llama、GLM、Qwen、DeepSeek、Mistral——2026 年你能叫出名字的现代大模型，几乎清一色用 RoPE（Rotary Position Embedding）。它不像正弦余弦编码那样把位置「加」到 embedding 上，而是用「旋转」把位置编码进 Q 和 K 向量，让注意力分数自然只依赖相对位置。本章从复数和旋转矩阵讲起，推导 RoPE 的核心公式，解释 θ 频率的设计，并延伸到长度外推的 NTK-aware 和 YaRN 方案。

阅读约 32 分钟 难度 ★★★★★ 前置：第 94、95、96 章

★ RoPE 要解决的核心问题

在动手推导前，先明确 RoPE 想干什么。它要找一个位置编码方式，满足一个关键性质：

目标：对于任意两个位置 m 和 n，注意力分数 ⟨q_m, k_n⟩ 应该只依赖于相对位置 m−n，而不单独依赖于 m 或 n。

这就是「相对位置编码」的诉求。前面讲过，相对位置比绝对位置更符合语言本质（「修饰语离中心词几步」比「修饰语在第几个位置」重要）。但传统相对位置编码（比如 T5 的偏置）做法是在注意力分数上加一个偏置项，每层每头都要，又贵又啰嗦。

RoPE 的精妙之处在于：它通过旋转 Q 和 K 向量，让它们的点积自动只依赖相对位置。不需要额外的偏置项，不改注意力公式，只改 Q、K 的生成方式。而且旋转操作可以无缝插进现有的注意力实现，工程友好。

位置 m 的 Q 向量 → 旋转角度 mθ
位置 n 的 K 向量 → 旋转角度 nθ
▼ 做点积
⟨q_m, k_n⟩ 只依赖 (m−n)θ —— 相对位置！
图 97-1 · RoPE 的核心思想：把位置变成旋转角度，点积自动出现相对位置

01 从复数和二维旋转开始

RoPE 的推导从最简单的二维情况入手。把 Q 向量的前两个分量 q = (q_0, q_1) 看作一个复数 q_0 + i·q_1。在复数世界里，「旋转一个角度 φ」就是乘以 e^(iφ) = cosφ + i·sinφ。

如果我们让位置 m 的 Q 旋转角度 mθ（θ 是固定的基础频率），即：

q_m = q · e^(imθ) = (q_0 + i·q_1) · (cos(mθ) + i·sin(mθ))

那么位置 m 的 Q 和位置 n 的 K 做点积（复数乘法取实部）：

⟨q_m, k_n⟩ = Re( q · e^(imθ) · conj(k · e^(inθ)) )
           = Re( q · conj(k) · e^(i(m−n)θ) )

关键来了：结果里出现了 e^(i(m−n)θ)，也就是结果只依赖 m−n 这个相对位置，完美达成目标！旋转角度 mθ 和 nθ 相减，绝对位置被消掉了，只剩下相对差。

💡 为什么旋转能消掉绝对位置

因为旋转是群操作：先转 mθ 再转 −nθ 等于转 (m−n)θ。Q 乘 e^(imθ)、K 乘 e^(inθ)，做点积时 K 取共轭（相当于反向旋转），两个旋转就合并成了相对旋转。这是 RoPE 数学之美的根源。

02 写成矩阵形式：分块旋转矩阵

复数乘法 e^(iφ) 等价于一个 2×2 旋转矩阵。把上面的复数操作翻译成矩阵语言，对于一对分量 (q_0, q_1)：

[ q_0' ]   = [ cos(mθ)  −sin(mθ)] [ q_0 ]
[ q_1' ]    [ sin(mθ)  cos(mθ)] [ q_1 ]

这就是把 (q_0, q_1) 这一对分量旋转角度 mθ。推广到 d 维向量，RoPE 把它两两配对，每对用一个不同频率 θ_i 的旋转：

q' = R_m · q，其中 R_m 是分块对角矩阵：

R_m = diag( R(mθ_0), R(mθ_1), ..., R(mθ_{d/2−1}) )

每个 R(mθ_i) = [ cos(mθ_i) −sin(mθ_i) ; sin(mθ_i) cos(mθ_i) ]

也就是说，d 维向量被拆成 d/2 个二维平面，每个平面独立旋转，旋转角度是 m × θ_i，θ_i 是该平面的基础频率。这和正弦余弦编码的多频率思想一脉相承——不同维度对用不同频率，近处靠高频、远处靠低频。

03 θ 频率怎么设计

θ_i 的取值沿用正弦余弦编码的思路：

θ_i = 10000^(−2i/d)   （i = 0, 1, ..., d/2−1）

也就是第 0 对维度频率最高（θ_0 = 1，转得最快），最后一对频率最低（θ ≈ 1/10000，转得最慢）。为什么这么设计：

  • 高频对（θ 大）：旋转快，相邻位置角度差异大，能分辨近距离（差 1、2 步）。
  • 低频对（θ 小）：旋转慢，几十几百步才转一圈，能分辨远距离。
  • 多尺度组合：和正弦余弦编码一样，覆盖从 1 到 10000 的尺度范围。

举个具体数字。假设 d=8（4 对维度），θ = [1, 1/100, 1/10000, ...]。位置 0 时所有角度都是 0（不旋转）；位置 1 时第 0 对转 1 弧度（约 57°）、第 1 对转 0.01 弧度（几乎没动）。这样每个位置的「旋转指纹」是唯一的，且近邻位置在高频对上差异明显。

04 代码：实现 RoPE

工业实现里有个高效技巧：不需要真的做矩阵乘法。RoPE 的旋转可以等价地用「逐元素乘以预计算的 cos、sin」实现，速度和内存都更优。

rope.py —— 旋转位置编码的高效实现
import torch
import torch.nn as nn
import math

def precompute_freqs_cis(dim, max_seq_len, base=10000.0):
    """预计算每个位置、每个频率对的 cos 和 sin。
    返回: (max_seq_len, dim//2) 的复数张量 e^(i*m*theta)
    """
    # 频率 theta_i = 1 / base^(2i/dim)
    freqs = 1.0 / (base ** (torch.arange(0, dim, 2).float() / dim))  # (dim//2,)
    t = torch.arange(max_seq_len).float()                            # (max_seq_len,)
    angles = torch.outer(t, freqs)                                   # (max_seq_len, dim//2)
    # 用复数表示，方便后面用 torch.view_as_complex
    freqs_cis = torch.polar(torch.ones_like(angles), angles)         # e^(i*m*theta)
    return freqs_cis

def apply_rotary_emb(x, freqs_cis):
    """对 x (batch, seq, n_head, dim) 应用 RoPE。
    把最后 dim 维当作 dim//2 个复数，逐个旋转。
    """
    d = x.shape[-1]
    # 把实数向量两两配对成复数: (..., d) -> (..., d/2) complex
    x_complex = torch.view_as_complex(x.float().reshape(*x.shape[:-1], d // 2, 2))
    # freqs_cis 广播到 head 维
    freqs_cis = freqs_cis.unsqueeze(0).unsqueeze(2)  # (1, seq, 1, d/2)
    # 复数乘法 = 旋转
    x_rotated = x_complex * freqs_cis
    # 转回实数: (..., d/2, 2) -> (..., d)
    x_out = torch.view_as_real(x_rotated).flatten(-2)
    return x_out.type_as(x)

# 测试
d = 8
seq_len = 5
freqs_cis = precompute_freqs_cis(d, seq_len)
q = torch.randn(1, seq_len, 4, d)  # batch=1, seq=5, n_head=4, dim=8
q_rot = apply_rotary_emb(q, freqs_cis)
print("旋转前后形状一致:", q.shape == q_rot.shape)  # True
print("位置 0 处未旋转:", torch.allclose(q[:, 0], q_rot[:, 0], atol=1e-6))  # True，m=0 角度为 0

这是 Meta Llama 官方实现的核心逻辑。关键技巧是用 torch.view_as_complex 把实数向量变复数，然后复数乘法天然就是旋转——比写循环做矩阵乘法快得多。

05 验证：注意力分数真的只依赖相对位置

口说无凭，我们做个实验。固定两个 query/key 向量，改变它们的绝对位置但保持相对距离，看注意力分数是否不变。

verify_relative.py —— 验证 RoPE 的相对位置性质
import torch
import math

def rope_rotate(vec, pos, dim):
    """对单个向量在位置 pos 做 RoPE 旋转（矩阵形式，便于理解）。"""
    result = vec.clone()
    for i in range(0, dim, 2):
        theta = pos / (10000 ** (i / dim))
        c, s = math.cos(theta), math.sin(theta)
        a, b = vec[i], vec[i+1]
        result[i]   = c * a - s * b
        result[i+1] = s * a + c * b
    return result

dim = 8
torch.manual_seed(0)
q = torch.randn(dim)
k = torch.randn(dim)

# 情况 A: q 在位置 10，k 在位置 13，相对距离 = 3
qA = rope_rotate(q, 10, dim); kA = rope_rotate(k, 13, dim)
scoreA = torch.dot(qA, kA).item()

# 情况 B: q 在位置 100，k 在位置 103，相对距离 = 3（绝对位置变了）
qB = rope_rotate(q, 100, dim); kB = rope_rotate(k, 103, dim)
scoreB = torch.dot(qB, kB).item()

print(f"相对距离 3，绝对位置 (10,13): 分数 = {scoreA:.6f}")
print(f"相对距离 3，绝对位置 (100,103): 分数 = {scoreB:.6f}")
print(f"两者差异: {abs(scoreA - scoreB):.2e}  (≈0 说明只依赖相对位置)")

运行后你会发现两个分数的差异在 1e-6 量级（浮点误差），几乎相等。这就证明了 RoPE 的相对位置性质——只要相对距离一样，绝对位置怎么变分数都不变。这是正弦余弦编码做不到的。

06 为什么旋转比相加好

对比 RoPE（旋转）和正弦余弦编码（相加），有几个本质区别：

对比维度正弦余弦（相加）RoPE（旋转）
作用对象embedding（输入层）Q、K（每层注意力）
位置信息形式绝对位置加性偏移旋转角度
注意力分数依赖绝对位置只依赖相对位置
外推性差较好（配合 NTK/YaRN 更好）
额外参数00（都是公式算的）
对 V 的影响有（V 也被加了 PE）无（只旋转 Q、K，不动 V）

最后一点很关键：RoPE 只旋转 Q 和 K，不旋转 V。因为注意力分数由 Q·K 决定，旋转 Q/K 就够让分数带上相对位置信息。V 是被加权求和的内容，旋转它没意义反而破坏语义。这是 RoPE 设计的精打细算。

07 外推难题：训练 4K，推理 32K 怎么办

RoPE 的相对位置性质让外推性好于绝对编码，但也不是无限外推。训练时见过位置 0~4096，推理时位置 10000 的旋转角度是训练时没见过的值，模型表现仍会下降。

于是出现了一系列「缩放 RoPE 频率」的外推技巧，核心思想是让旋转角度的增长变慢，让训练范围内见过的角度模式能覆盖更长的推理范围。

方法一：位置线性插值（PI）

最直接的做法：把推理时的位置 m 缩放成 m × (L_train / L_infer)。比如训练 4K、推理 32K，就把所有位置除以 8。这样 32K 的位置被压缩回 4K 范围，落进训练见过的角度区间。

位置插值：m' = m × (L_train / L_infer)，然后用 m' 算旋转角度。

缺点：相邻位置的旋转角度也被压缩了，导致近距离的分辨能力下降——模型对「相邻 token」的位置感知变钝。

方法二：NTK-aware 缩放

NTK（Neural Tangent Kernel）理论的启发：不要均匀缩放所有频率，而是主要压低频、保留高频。具体做法是调整 base：

base' = base × (scale)^(d/(d−2))，其中 scale = L_infer / L_train

这样高频对（θ 大）几乎不变，近距离分辨力保留；低频对（θ 小）波长变长，覆盖更远距离。NTK-aware 是目前最常用的外推方法之一，效果比纯位置插值好。

方法三：YaRN

YaRN 是 NTK 的进阶版。它发现不同频率维度对插值的敏感度不同，于是按频率分段：高频段不插值（保留近距离）、低频段做位置插值（扩展远距离）、中间段做平滑过渡。YaRN 是目前外推效果最好的方法之一，Llama 3、Qwen2 的长上下文版本都用它。

📌 这些方法都需要少量微调

纯外推（不做任何训练）效果都有限。实践中，改了 RoPE 的 base 或插值方式后，通常要在长文本上做少量继续预训练（几十亿 token），让模型适应新的角度分布。这就是为什么 GLM-4-Long、Llama-3-128K 都标明「在长文本上微调过」。

08 代码：实现 NTK-aware 频率缩放

ntk_rope.py —— NTK-aware 的频率计算
import torch
import math

def compute_rope_freqs(dim, max_seq_len, base=10000.0, scale=1.0, method="ntk"):
    """计算 RoPE 频率，支持原始 / 位置插值 / NTK-aware 三种模式。
    scale = L_infer / L_train，比如训练 4K 推理 16K 则 scale=4。
    """
    if method == "base":
        # 原始 RoPE
        freqs = 1.0 / (base ** (torch.arange(0, dim, 2).float() / dim))
    elif method == "pi":
        # 位置插值：均匀缩放所有频率
        freqs = 1.0 / (base ** (torch.arange(0, dim, 2).float() / dim))
        freqs = freqs / scale
    elif method == "ntk":
        # NTK-aware：主要压低频，通过调 base 实现
        base_scaled = base * (scale ** (dim / (dim - 2)))
        freqs = 1.0 / (base_scaled ** (torch.arange(0, dim, 2).float() / dim))

    t = torch.arange(max_seq_len).float()
    angles = torch.outer(t, freqs)
    return torch.polar(torch.ones_like(angles), angles), freqs

# 对比三种方法在 dim=8 时的最低频（决定远距离分辨力）
for method in ["base", "pi", "ntk"]:
    _, freqs = compute_rope_freqs(8, 100, scale=4, method=method)
    print(f"{method:5s}: 最高频={freqs[0]:.4f}, 最低频={freqs[-1]:.6f}")
# ntk 的最高频几乎不变（近距离保留），最低频明显变小（远距离扩展）

运行后你会看到：原始 RoPE 最高频 1.0、最低频 0.0001；位置插值把两者都除以 4；NTK 则最高频几乎不变、最低频变得更小。这正是「保留近距离、扩展远距离」的体现。

09 为什么 RoPE 成了事实标准

回顾 RoPE 的优势，就明白它为什么能统一江湖：

🎯

相对位置，天然贴合语言

注意力分数只依赖 m−n，符合「修饰语离中心词几步」这种语言本质需求。

⚙️

工程友好，零额外参数

只改 Q、K 的生成，不动注意力公式，不加参数，实现就几十行。可无缝插入任何注意力。

📈

外推可调，支持长上下文

配合 NTK/YaRN，可以从 4K 扩展到 32K、128K 甚至百万 token，是长上下文时代的基石。

🔌

和 KV Cache 完美兼容

每个位置的旋转角度只依赖自己，推理时算一次就固定，存进 KV cache 不用重算。

这四点合起来，让 RoPE 在 2023 年后迅速取代了正弦余弦编码、可学习 PE、T5 相对偏置等所有前辈。Llama 2 开始全面采用，之后几乎所有开源模型跟进。2026 年，RoPE 就是位置编码的默认答案。

⚠️ RoPE 不是万能的

RoPE 也有局限：①超长外推仍需微调，纯外推性能衰减；②旋转操作在高维时数值精度敏感，低精度量化（如 INT4）可能丢失位置信息；③θ 频率和 base 的选择需要随模型规模调整，没有「一刀切」的最优值。这些是当前长上下文研究的活跃方向。

10 本章小结

RoPE 这一章信息量大，核心要点梳理：

RoPE 的数学核心
  • 把位置 m 编码成旋转角度 mθ
  • 对 Q、K 的每对维度做旋转 R(mθ_i)
  • θ_i = 10000^(−2i/d)，多频率设计
  • 点积结果只依赖 m−n（相对位置）
外推方案演进
  • 位置插值 PI：均匀压缩，损近距离
  • NTK-aware：主要压低频，保近距离
  • YaRN：分段插值，效果最佳
  • 都需要少量长文本微调
