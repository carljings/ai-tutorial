LLM 面试题全集
50 道高频题与深度解答

2026 年 AI 岗位面试考什么？本章整理 50 道高频面试题，覆盖原理、工程、系统设计、场景题四大类，每题给出标准答案 + 加分点 + 常见误区。

阅读约 50 分钟难度 ★★★★面试必备

01 原理题（15 道）

Q1: 为什么 Attention 要除以 √dₖ？

答：Q·K 点积的方差随维度 dₖ 线性增长。dₖ 大时，点积绝对值大，softmax 进入饱和区（梯度趋零）。除以 √dₖ 把方差拉回 1，让 softmax 工作在合理区间。加分：能推导方差为 dₖ。误区：说「防止数值溢出」——那只是表象，根因是梯度消失。

Q2: KV Cache 为什么能加速？占多少显存？

答：Attention 的 K、V 只取决于已有 token，新 token 不改变它们。缓存后每步只算新 token 的 Q。显存 = 2×n_layers×n_heads×head_dim×seq_len×batch×dtype_bytes。加分：能算具体数字（如 80 层 128K 上下文 fp16 ≈ 数百 GB）。

Q3: MoE 怎么做到「大容量低计算」？

答：MoE 把 MLP 替换为多个专家+路由器。每个 token 只激活 Top-K 个专家（如 2/8），总参数大但单次计算量小。路由器用负载均衡损失防止只用少数专家。

Q4: RoPE 为什么支持长上下文外推？

答：RoPE 把相对位置编码进 Q/K 的旋转。它的数学性质使得 <q_m, k_n> 只依赖 m-n（相对位置），而非绝对位置。配合 NTK-aware 缩放，可以从训练长度外推到更长。

Q5: 交叉熵损失和困惑度的关系？

答：困惑度 PPL = exp(平均交叉熵)。PPL=10 意味着模型在每个位置像在 10 个候选词间犹豫。PPL 越低越好。

Q6: SGD、Momentum、Adam 的区别？

答：SGD 纯梯度；Momentum 加指数移动平均（惯性）；Adam 同时维护梯度和梯度平方的移动平均（自适应学习率）。AdamW 是大模型标配（Adam+权重衰减）。

Q7: SFT 和 RLHF 解决什么不同问题？

答：SFT 教「格式和指令遵循」（会对话）；RLHF 教「质量和偏好」（答得好）。SFT 后的模型会回答但可能不好，RLHF 让它更好。

Q8: DPO 比 RLHF 简单在哪？

答：DPO 跳过显式奖励模型和 PPO 强化学习，直接从偏好数据推导最优策略。一个损失函数搞定，更稳定、更省资源。

Q9: 为什么训练初期要多加代码数据？

答：代码数据训练模型的结构化推理能力（逻辑、因果关系）。早期注入塑造模型的「思维骨架」，这也是 GLM-5 Agentic Coding 强的基因。

Q10: Model Collapse 是什么？怎么防？

答：用 AI 生成数据再训 AI，多代后分布收窄（多样性丧失）。防护：始终混入真实数据、多样性追求、质量过滤、分布漂移监控。

Q11: FlashAttention 为什么快？

答：把 Attention 切块在 SRAM（片上缓存）完成计算，避免读写 HBM 中的 T×T 中间矩阵。计算量不变但 IO 大幅减少。

Q12: 推测解码为什么无损？

答：小模型猜的 token，大模型逐一验证。只接受大模型也认可的。最终输出与纯大模型完全一致，只是更快（猜对了就省时间）。

Q13: 温度采样中 T 的数学效果？

答：softmax(z/T)。T→0 趋近 argmax（贪心）；T→∞ 趋近均匀（完全随机）；T=1 原始分布。T 控制「创造性 vs 确定性」。

Q14: BPE 怎么处理没见过的词？

答：字节级 BPE 把文本编码成 UTF-8 字节，再在字节层做 BPE。任何字符都能还原到 256 个字节，永不 OOV。

Q15: Scaling Laws 的实践含义？

答：Loss 随参数/数据/算力幂律下降，可预测。Chinchilla 进一步：最优 D/N≈20（数据量应为参数数的 20 倍）。指导「该用多少数据训多大的模型」。

02 工程题（15 道）

Q16: 什么是 Harness？为什么 2026 年这么火？

答：Harness 是包裹模型的工程外壳（权限/Hook/上下文/反馈/成本）。火的原因：模型趋同，差异在工程；智能体能力解锁需要 Harness；开源基础设施成熟。

Q17: Agent Loop 的七种死法和防护？

答：死循环（重复检测）、工具异常（重试+喂回错误）、上下文爆炸（压缩+子Agent）、任务漂移（定期重申目标）、权限越界（明确边界）、预算失控（上限）、格式错误（结构化校验）。

Q18: PEV 环是什么？为什么重要？

答：Prompt-Execute-Verify。模型生成→执行→验证→反馈修正。把「一次猜对」变成「反复试错到成功」。编程 Agent 成熟快是因为代码可自动验证。

Q19: RAG 的分块为什么决定 70% 效果？

答：分块决定检索粒度。切不好→语义断裂/主题混杂/元数据丢失。好的分块让语义相同的内容聚合，不同内容分开。

Q20: 混合检索为什么比纯向量好？

答：向量擅长语义匹配（「开心」≈「快乐」），BM25 擅长精确匹配（专有名词/代码）。RRF 融合两者，取长补短。

Q21: Rerank 为什么有效？

答：Embedding 为快把整段压成一个向量，信息损失大。Rerank 用 Cross-Encoder 精细计算 query-文档相关性，精度高得多。典型：召回 50→Rerank 精选 5。

Q22: MCP 解决什么问题？

答：工具调用格式碎片化（OpenAI/智谱/LangChain 各一套）。MCP 统一接口，写一次工具，所有客户端通用。号称「AI 工具的 USB-C」。

Q23: 权限模型的 allow/ask/deny 三态？

答：allow 自动放行（只读）、ask 需确认（写操作）、deny 绝对禁止（删/push）。默认拒绝（最小权限），deny 优先匹配。

Q24: 上下文管理的三层记忆？

答：工作记忆（当前上下文窗口）、短期记忆（会话摘要）、长期记忆（向量库/RAG）。子 Agent 实现上下文隔离。

Q25: 怎么防止 Agent 的间接注入？

答：纵深防御——指令隔离（标签区分可信指令和不可信数据）+ 权限最小化 + 外发审计 Hook + 高危操作人工确认。

Q26: Prompt Caching 怎么省钱？

答：固定不变的 prompt 前缀（系统提示/技能说明）缓存，后续请求只算新增部分。对长系统提示可省 50-90%。

Q27: 怎么选 RAG vs 微调？

答：加知识→RAG（秒更新/可溯源）；改行为风格→微调；两者常组合。

Q28: vLLM 为什么吞吐量高？

答：PagedAttention（分页 KV，显存利用率 95%）+ 连续批处理（请求动态进出）+ Prefix Caching。三者叠加，吞吐量比朴素高 3-10 倍。

Q29: INT4 量化损失多大？

答：约 2-3% 精度损失，但显存降到 1/4，速度快 2×。ChatGLM-6B INT4 让 6B 模型跑在 6GB 显存上。

Q30: 子 Agent 怎么实现上下文隔离？

答：派一个独立 Agent，用全新 messages 处理子任务，完成后只把结论返回给主 Agent。子任务的中间过程不进主 Agent 上下文。

05 面试技巧

✅ 加分行为
  • 用具体数字（不是「很多」是「95%」）
  • 说「为什么」而非「是什么」
  • 主动画架构图
  • 聊到 trade-off 展示深度
  • 提你的实战经验
❌ 减分行为
  • 只会背概念不懂原理
  • 说「用 AI 解决」但不说怎么解决
  • 忽视 trade-off
  • 不问澄清问题
  • 夸大经验
