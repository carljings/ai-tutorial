多头注意力
为什么「多看几个角度」更有效

到目前为止我们讲的都是单头注意力：一组 Q/K/V 投影，算出一个注意力输出。但真实 Transformer 从来不用单头——GPT-3 用 96 头，GLM-4 用 32 头。为什么不把钱花在一个「大号单头」上，而要拆成多个小头？本章讲清多头的并行计算原理、头数的维度分配、不同头学到不同语义关系的原因，以及头数选择的经验法则。

阅读约 26 分钟 难度 ★★★★ 前置：第 94-97 章

★ 单头的局限

先把单头注意力的算式摆出来。给定输入 X (n × d_model)，单头注意力做：

Q = X·Wq, K = X·Wk, V = X·Wv（投影到 d_k / d_v 维）
Attention(X) = softmax(Q·K^T / √d_k) · V · Wo

这里的 Wq、Wk、Wv、Wo 是四套参数。问题是：一组投影，只能学到一种注意力模式。比如它可能学会了「主语关注谓语」，就很难同时再学好「代词关注指代对象」「形容词关注名词」。

⚠️ 单头的瓶颈

注意力权重的 softmax 是一个竞争性分配：每个 token 把 100% 的关注度分给所有 token。如果某个关系（比如「主谓」）占用了大头权重，其他关系（比如「指代」「修饰」）就分不到多少。一组投影难以同时表达多种并行的语义关系。

举个具体例子。句子「穿红衣服的女孩喂那只猫」。单头注意力要么重点抓「女孩-喂」的主谓关系，要么重点抓「那只-猫」的指代关系，很难两个都满分——因为 softmax 的预算是固定的。但现实中这两件事要同时发生，才能正确理解句子。

01 多头的核心思想：并行多视角

多头注意力的解法很直接：搞好几组投影，每组算一个独立的注意力，最后拼起来。每组叫一个「头（head）」。

MultiHead(X) = Concat(head_1, head_2, ..., head_h) · Wo

其中每个 head_i = Attention(X·Wq_i, X·Wk_i, X·Wv_i)

也就是说，h 个头，每个头有自己独立的 Wq_i、Wk_i、Wv_i，算出独立的注意力输出，最后把 h 个输出在特征维拼接，过一个输出投影 Wo 回到 d_model。

→ d_k 维

把单头想象成一个全科医生，什么病都看，但每个病都看得不深。多头就是一个专家团队：眼科专家专门看眼睛、心脏专家专门看心脏、神经科专家专门看神经。每人专注一个方面，最后把各位专家的意见汇总，诊断质量远高于一个全科医生。每个头就是一个「语义关系专家」。

02 维度怎么分：保持总计算量不变

一个关键的设计巧思：多头不是把计算量翻 h 倍。原论文的做法是让每个头的维度 d_k = d_model / h，这样 h 个头拼接起来正好是 d_model，总参数量和单头差不多。

以 d_model=512、h=8 为例：

  • 单头：Wq 是 512×512，参数 262144；
  • 多头（8 头）：每个头 Wq_i 是 512×64，参数 32768，8 个头加起来 32768×8 = 262144。

参数总数一样！计算量也一样（h 个小矩阵乘 vs 一个大矩阵乘）。所以多头不是用更多算力换更多能力，而是用同样的算力，换更好的表达结构——把一个大矩阵拆成 h 个小矩阵，每个小矩阵可以专注学一种关系。

经验法则：每头维度 d_k 通常取 64 或 128。d_model = h × d_k。比如 d_model=4096、d_k=128，则 h=32。模型规模变大时，往往优先加宽 d_model，相应地增加头数 h，保持 d_k 在 64~128 这个「甜点」区间。

03 并行计算：一次矩阵乘搞定所有头

实现多头时，不会真的写 h 个独立的注意力循环——那样太慢。工程上的高效做法是把所有头「折叠」进 batch 维，用一次矩阵乘法同时算完所有头。

具体来说：①把 Wq、Wk、Wv 写成一个大矩阵（d_model × d_model），一次投影得到所有头的 Q/K/V；②把 (batch, seq, d_model) reshape 成 (batch, seq, h, d_k) 再转置成 (batch, h, seq, d_k)，把 h 当成 batch 维；③做一次注意力，所有头并行算；④reshape 回 (batch, seq, d_model)，过 Wo。

04 代码：完整的多头注意力实现

multihead_attention.py —— 可运行的多头注意力
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, num_heads):
        super().__init__()
        assert d_model % num_heads == 0, "d_model 必须能被 num_heads 整除"
        self.d_model = d_model
        self.num_heads = num_heads
        self.d_k = d_model // num_heads  # 每头维度

        # 四个投影矩阵（合并所有头到一个大矩阵）
        self.W_q = nn.Linear(d_model, d_model, bias=False)
        self.W_k = nn.Linear(d_model, d_model, bias=False)
        self.W_v = nn.Linear(d_model, d_model, bias=False)
        self.W_o = nn.Linear(d_model, d_model, bias=False)

    def forward(self, x, mask=None):
        B, S, _ = x.shape
        # 1. 一次投影出所有头的 Q/K/V
        Q = self.W_q(x)  # (B, S, d_model)
        K = self.W_k(x)
        V = self.W_v(x)

        # 2. reshape: (B, S, d_model) -> (B, S, h, d_k) -> (B, h, S, d_k)
        Q = Q.view(B, S, self.num_heads, self.d_k).transpose(1, 2)
        K = K.view(B, S, self.num_heads, self.d_k).transpose(1, 2)
        V = V.view(B, S, self.num_heads, self.d_k).transpose(1, 2)

        # 3. 注意力（所有头并行，h 当 batch）
        scores = Q @ K.transpose(-2, -1) / math.sqrt(self.d_k)  # (B, h, S, S)
        if mask is not None:
            scores = scores.masked_fill(mask == 0, float('-inf'))
        weights = F.softmax(scores, dim=-1)
        out = weights @ V  # (B, h, S, d_k)

        # 4. 拼回: (B, h, S, d_k) -> (B, S, h, d_k) -> (B, S, d_model)
        out = out.transpose(1, 2).contiguous().view(B, S, self.d_model)

        # 5. 输出投影
        return self.W_o(out), weights

# 测试
torch.manual_seed(0)
mha = MultiHeadAttention(d_model=64, num_heads=8)
x = torch.randn(2, 10, 64)  # batch=2, seq=10, d_model=64
out, w = mha(x)
print("输出形状:", out.shape)       # (2, 10, 64)
print("权重形状:", w.shape)         # (2, 8, 10, 10) —— 每头一个 10×10 矩阵
print("每头权重行和:", w[0, 0, 0].sum().item())  # 1.0

关键看 view + transpose 那两步：它把 d_model 拆成 (h, d_k)，再把 h 维换到 batch 位置。这样后面 Q @ K.transpose 会自动对每个头独立做矩阵乘——GPU 并行，一次算完。这是所有 Transformer 实现的标准套路。

05 不同的头学了什么

多头注意力最迷人的发现：训练完后，不同的头会自发地专注于不同的语法/语义关系。这是「可解释性」研究的经典成果（著名的「Attention is not Explanation」系列论文讨论过）。

对训练好的模型做注意力可视化，会看到这样的分工：

头的类型学到的模式典型表现
指代头代词 → 它指代的名词"它"关注"猫"
句法头主语 → 谓语，动词 → 宾语"猫"关注"追"
修饰头形容词 → 名词"红色的"关注"苹果"
位置头关注前一个/后一个 token固定看 i−1 或 i+1
停用词头几乎只关注 BOS 或句号权重集中在一个固定位置

这种「自发分工」是多头注意力的核心价值。单头做不到——它只有一组投影，只能学一种模式。多头相当于给模型提供了一个「关系字典」的并行插槽，每个头认领一种关系，组合起来才能处理语言里错综复杂的结构。

📌 不是所有头都重要

研究也发现，多头里有一部分是「冗余头」或「停用词头」——它们的注意力几乎固定不动，删掉对效果影响很小。这也是后来 MQA/GQA（下一章）能成功的依据：既然很多头在做相似的事，那共享一部分投影（K/V）也不会损失太多。但核心的语义头是不可替代的。

06 头数怎么选

头数 h 是个重要超参。太少不够，太多也不行。经验法则：

  • 太少（h=1、2）：表达力不足，学不到多种关系。早期小模型也至少用 4~8 头。
  • 甜点区间（h=8~32）：大多数模型的选择。每头 d_k 在 64~128，参数效率高。
  • 太多（h=64+）：每头维度太小（d_k < 16），单头表达力太弱，反而下降。GPT-3 用 96 头是因为它 d_model=12288 极大，96 头每头仍有 128 维。

一个被反复验证的经验：固定 d_model，改变 h，存在一个最优值。h 太小单头太宽学不出多样关系，h 太大单头太窄表达力不够。这个最优值通常让 d_k 落在 64~128。

07 实验：多头 vs 单头的参数等价性

我们用代码验证「多头和单头参数数相同」这件事，并对比它们在简单任务上的表现差异。

count_params.py —— 对比参数量
import torch.nn as nn

# 单头：d_model=64，投影到 64 维
class SingleHead(nn.Module):
    def __init__(self, d=64):
        super().__init__()
        self.q = nn.Linear(d, d, bias=False)
        self.k = nn.Linear(d, d, bias=False)
        self.v = nn.Linear(d, d, bias=False)
        self.o = nn.Linear(d, d, bias=False)

# 多头：d_model=64，8 头每头 8 维，但实现上仍是 64×64 大矩阵
# （MultiHeadAttention 类见上一段代码）
single = SingleHead(64)
multi = MultiHeadAttention(64, 8)

def count(m): return sum(p.numel() for p in m.parameters())

print(f"单头参数: {count(single)}")   # 4 × 64 × 64 = 16384
print(f"多头参数: {count(multi)}")    # 16384 —— 完全一样

确认：参数量一模一样。但多头在多数任务上效果更好，因为它把参数结构化了——8 个独立的 8 维子空间，比一个 64 维大空间更容易学到多样化的模式。

08 从 MHA 到 MQA、GQA 的预告

标准多头注意力（MHA）有个推理时的痛点：每个头都有独立的 K、V，推理时要为每个头存 KV cache。头数多、上下文长时，KV cache 的显存占用惊人（第 100 章专题）。

于是出现两个变体：

  • MQA（Multi-Query Attention）：所有头共享一组 K、V，只有 Q 是多头的。KV cache 直接除以 h。
  • GQA（Grouped-Query Attention）：MHA 和 MQA 的折中，把头分组，组内共享 K、V。Llama 2/3 用的就是 GQA。

这些是推理优化的关键技巧，下一章详细讲。这里只需记住：多头的「每头独立 K/V」是推理显存的大头，值得专门优化。

⚠️ 多头不是免费午餐

多头虽然总参数不变，但有几个隐藏成本：①KV cache 随头数线性增长（推理显存）；②reshape 和 transpose 操作在 GPU 上有内存布局开销；③头间没有信息交流（直到 Wo 才合并），可能学到冗余模式。这些是 GQA、Flash Attention 等技术要解决的。

09 用 PyTorch 内置的 MultiheadAttention

实际开发不用自己写，PyTorch 提供了优化过的内置实现：

pytorch_mha.py —— 用内置 MultiheadAttention
import torch
import torch.nn as nn

# PyTorch 内置的多头注意力（已优化，支持 Flash Attention 后端）
mha = nn.MultiheadAttention(
    embed_dim=512,
    num_heads=8,
    batch_first=True,   # 输入 (batch, seq, dim) 而非 (seq, batch, dim)
    bias=False,
)

x = torch.randn(4, 20, 512)        # batch=4, seq=20, d_model=512
out, weights = mha(x, x, x)        # self-attention: Q=K=V=x
print("输出:", out.shape)           # (4, 20, 512)
print("权重:", weights.shape)       # (4, 20, 20) —— 平均了所有头
# 注意：内置版默认返回的 weights 是所有头的平均，不是每头一个

工业代码里更常见的是直接用 torch.nn.functional.scaled_dot_product_attention，它底层会自动选择最优后端（Flash Attention、Memory-Efficient、Math），性能最好。

10 本章小结

多头的核心价值
  • 单头只能学一种注意力模式，softmax 预算竞争
  • 多头 = h 组独立投影，并行学多种语义关系
  • 训练后不同头自发分工：指代、句法、修饰...
  • 总参数和单头相同（d_k = d_model/h）
