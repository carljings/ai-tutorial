金融是 AI 落地最成熟的行业之一，因为数据量大、标注清晰、试错成本高（每一笔坏账都是真金白银）。我们以一家持牌消金公司的智能风控系统为例。

1.1 业务痛点

传统风控依赖征信报告 + 规则引擎，对「信用白户」（没有信贷记录的人）几乎束手无策。而这一群体在中国有数亿人，是消金公司必争的市场。AI 要解决的问题是：用替代数据（行为数据、社交数据、设备数据）预测这类用户的违约概率。

1.2 技术架构

这套系统不是一个大模型搞定，而是三层架构：规则层（硬规则拦截明显欺诈）、机器学习层（XGBoost + LightGBM 做违约概率打分）、大模型层（对非结构化数据做特征提取和语义分析）。

# 信贷风控的 LLM 应用：对用户填写的职业信息做归一化和风险信号提取
from openai import OpenAI
import json

client = OpenAI()

def extract_risk_signals(application_text: str) -> dict:
    """从用户申请文本里提取结构化风险信号。"""
    prompt = f"""你是信贷风控分析师。从以下贷款申请信息中提取风险信号。

申请信息：{application_text}

请输出 JSON，包含字段：
- profession_normalized: 归一化后的职业类别（如"外卖骑手"→"物流配送"）
- income_credibility: 收入可信度（high/medium/low），判断依据是否合理
- risk_keywords: 识别到的风险关键词列表（如"现金结算"、"多份兼职"）
- fraud_signal: 是否有欺诈信号（true/false）及原因"""

    resp = client.chat.completions.create(
        model="gpt-4o",
        response_format={"type": "json_object"},
        messages=[{"role": "user", "content": prompt}],
        temperature=0,
    )
    return json.loads(resp.choices[0].message.content)

# 真实案例数据（脱敏）
sample = "我是做外卖配送的，月收入8000-12000，有时也跑闪送，现金结算为主"
result = extract_risk_signals(sample)
print(json.dumps(result, ensure_ascii=False, indent=2))
# 输出示例：
# {
#   "profession_normalized": "物流配送",
#   "income_credibility": "medium",  // 收入波动大
#   "risk_keywords": ["现金结算", "多份兼职"],
#   "fraud_signal": {"flag": false, "reason": "信息合理但需核实收入流水"}
# }

1.3 业务效果

这套系统上线后，对信用白户的违约识别准确率提升了 18%，坏账率从 4.2% 降到 3.5%。关键是审批时间从平均 8 小时压缩到 3 分钟——这是纯人力风控永远做不到的。但要注意：金融场景对可解释性要求极高，LLM 的黑盒输出不能直接用于拒贷决策，它只能作为特征输入给下游的评分卡模型。

★ 二、医疗：影像辅助诊断与病历结构化

医疗 AI 的核心原则是「辅助不替代」——AI 给建议，医生做决策。这个原则在合规上不可逾越。我们看一个三甲医院的落地案例。

2.1 场景一：CT 影像肺结节筛查

放射科医生每天要看上百张 CT，漏诊小结节是常见问题。AI 辅助筛查系统用 CNN（通常是 ResNet 或 UNet 的变体）做结节检测，把可疑位置标注出来给医生复核。

# 肺结节检测的简化推理流程（实际部署用 ONNX Runtime 加速）
import numpy as np

class NoduleDetector:
    def __init__(self, model_path):
        # 加载训练好的 3D UNet 模型
        self.session = load_onnx_model(model_path)

    def detect(self, ct_volume: np.ndarray) -> list:
        """
        ct_volume: (depth, height, width) 的三维 CT 数据
        返回：结节位置列表 [{x, y, z, diameter_mm, confidence}]
        """
        # 预处理：窗宽窗位调整、归一化
        volume = apply_lung_window(ct_volume)
        volume = normalize(volume)

        # 模型推理，输出每个体素是结节的概率图

2.2 场景二：电子病历结构化

医生的病历大多是自由文本（「患者主诉头痛三天，既往高血压病史十年」），没法直接做统计分析。LLM 在这里的杀手级应用是把自由文本转成结构化字段。

# 病历结构化：自由文本 → 结构化 JSON
def structure_medical_record(emr_text: str) -> dict:
    prompt = f"""从以下病历文本中提取结构化信息，严格按 JSON 输出。

病历：{emr_text}

提取字段：
- chief_complaint: 主诉
- present_illness: 现病史（含发病时间、症状演变）
- past_history: 既往史（疾病列表，每个含名称和病程）
- medications: 当前用药（名称+剂量+频次）
- allergies: 过敏史
- vital_signs: 生命体征（血压、心率、体温，有则提取）"""

    resp = client.chat.completions.create(
        model="gpt-4o",
        response_format={"type": "json_object"},
        messages=[{"role": "user", "content": prompt}],
        temperature=0,
    )
    return json.loads(resp.choices[0].message.content)

record = structure_medical_record(
    "患者男，56岁，主诉反复胸闷气促3年，加重1周。"
    "既往高血压病史10年，长期口服氨氯地平5mg qd。否认药物过敏史。"
    "查体：BP 150/95mmHg，HR 88次/分，T 36.5℃。"
)
# 准确率：关键字段（主诉、既往史、用药）抽取准确率 95%+

这个应用的价值在于：原本需要护士花 20 分钟手工录入的结构化数据，现在 10 秒完成，错误率更低，而且能直接进入医院的科研数据库做后续分析。

★ 三、教育：个性化学习路径与智能答疑

教育行业的 AI 落地分两块：To C 的学习类应用（拍搜、AI 老师）和 To B 的教学管理平台。我们重点看「AI 答疑老师」这个场景——它最能体现大模型的能力边界。

3.1 痛点与架构

学生在做作业时遇到不会的题，传统方案是搜题库（必须有原题）或问老师（响应慢）。AI 答疑老师要做到：理解题目、给出解题步骤、针对学生的薄弱点做个性化讲解。技术上是「RAG + Agent + 多模态」的组合。

# AI 数学答疑老师的核心 Agent 逻辑
class MathTutorAgent:
    def __init__(self):
        self.knowledge_base = load_textbook_kb()      # 教材知识点库
        self.similar_problems = load_problem_db()     # 题库

    def solve(self, question: str, student_level: str = "初二") -> dict:
        # 第 1 步：识别题目涉及的知识点
        concepts = self.identify_concepts(question)
        # 第 2 步：从教材库召回相关知识点讲解
        refs = self.retrieve_refs(concepts)
        # 第 3 步：根据学生水平生成个性化讲解
        solution = self.generate_solution(question, refs, student_level)
        # 第 4 步：推荐 2-3 道同类题巩固
        practices = self.recommend_practices(concepts)
        return {
            "concepts": concepts,
            "solution": solution,
            "practice_problems": practices,
        }

    def generate_solution(self, question, refs, level):
        prompt = f"""你是{level}的数学老师，请用启发式方式讲解这道题。

题目：{question}
相关知识点：{refs}

要求：
1. 不要直接给答案，先引导学生思考关键步骤
2. 每一步都要解释「为什么这么做」
3. 最后给出完整解答和答案验证
4. 语言要符合{level}学生的认知水平"""
        return call_llm(prompt)

3.2 关键效果与教训

这个系统在试点学校的真实数据：学生平均作业完成时间下降 35%，同类题错误率下降 40%。但有两个教训值得注意：

  • 必须做答案校验：LLM 偶尔会算错（尤其多步计算），必须接一个符号计算引擎（如 SymPy）做结果校验，错了就重新生成。
  • 讲解比答案更重要：家长最怕 AI 直接给答案（学生抄作业）。所以 prompt 设计要强制「先引导后解答」，并在产品层面限制直接答案输出。

★ 四、制造：工业视觉质检与设备预测性维护

制造业的 AI 落地相对低调，但价值巨大。一块芯片、一个手机外壳的良率每提升 1%，对大厂就是几千万的利润。我们看两个场景。

4.1 场景一：手机外壳表面缺陷检测

传统的人工质检，一个工人盯着流水线 8 小时，漏检率在 3-5%。而且人眼疲劳后准确率骤降。AI 视觉质检用工业相机 + 深度学习模型，能做到 24 小时稳定运行。

# 工业缺陷检测的难点：缺陷样本少，需要用小样本学习
import torch
from transformers import AutoModelForImageSegmentation

class SurfaceDefectDetector:
    def __init__(self):
        # 实际部署：在工厂产线用 TensorRT 加速的轻量模型
        # 这里演示用预训练模型做迁移学习
        self.model = AutoModelForImageSegmentation.from_pretrained(
            "factory-defect-v2"  # 在 5000 张标注样本上微调过
        )

    def inspect(self, image) -> dict:
        """检测手机外壳表面的划痕、凹陷、色差。"""
        with torch.no_grad():
            output = self.model(image)
        defects = parse_output(output)
        return {
            "has_defect": len(defects) > 0,
            "defects": [
                {"type": d.type, "area_px": d.area, "severity": d.severity}
                for d in defects
            ],
            "pass_rate": calculate_pass_rate(),
        }

# - 漏检率：从 4.2% 降到 0.3%（关键改善）
# - 误检率：1.5%（可接受，人工复核即可）

4.2 场景二：设备预测性维护

工厂里的数控机床、注塑机一旦意外停机，一条产线每小时损失几万到几十万。预测性维护用传感器数据（振动、温度、电流）提前预测故障。这里 LLM 的角色是「生成维护报告」——把时序模型的预测结果翻译成工人能看懂的维护建议。

# 预测性维护的报告生成
def generate_maintenance_report(sensor_data: dict, anomaly_score: float) -> str:
    prompt = f"""你是工厂设备维护工程师。根据以下传感器数据和异常评分，生成维护建议报告。

设备：CNC 加工中心 #A12
传感器数据（最近24小时）：{sensor_data}
异常评分：{anomaly_score}（0-1，超过 0.7 建议干预）

报告要求：
1. 异常原因初步判断（基于振动频率、温度趋势）
2. 建议措施（立即停机/降级运行/下次保养时处理）
3. 需要准备的备件清单
4. 预计维修耗时"""
    return call_llm(prompt)

★ 五、跨行业共性经验

看完这四个案例，有几个跨行业的共性经验值得提炼：

  • AI 从不单独存在：每个落地的 AI 系统都是「规则 + 传统 ML + 大模型」的混合架构。指望一个 LLM 解决所有问题是不现实的。
  • 数据质量决定上限：金融的标注数据、医疗的脱敏数据、制造的缺陷样本，哪个行业的数据壁垒高，哪个行业的 AI 价值就大。
  • 可解释性是刚需：尤其金融和医疗，黑盒模型必须配合可解释的决策路径，否则过不了合规关。
  • 从辅助到自主是渐进过程：先做 AI 辅助人（低风险），验证效果后再逐步扩大 AI 的自主决策范围。

★ 本章小结

这一章我们横跨四个行业看了 AI 的真实落地。核心要点：

  • 金融：风控 + 反欺诈是主战场，LLM 用于非结构化数据特征提取，配合传统评分卡模型，坏账率显著下降。
  • 医疗：影像辅助诊断（CNN）+ 病历结构化（LLM）双轮驱动，核心原则是「辅助不替代」，必须做答案校验。
  • 教育：AI 答疑老师的价值在个性化讲解而非给答案，必须接符号计算引擎兜底，防止算错。
  • 制造：工业视觉质检（降漏检）+ 预测性维护（降停机），LLM 的角色是把模型输出翻译成工人能懂的建议。
  • 共性经验：混合架构、数据为王、可解释性、渐进自主——这是所有行业 AI 落地的通用法则。
