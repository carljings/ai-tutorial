推理模型深度
「想得更久」如何变成「答得更对」

2025-2026 年最重要的突破之一是推理模型——o1、DeepSeek-R1、GLM 推理增强。它们用更多推理时计算换取更高准确率。本章深入思维链、RLVR、Test-time Compute 的原理与实现，讲清「推理模型为什么强」。

阅读约 35 分钟 难度 ★★★★★ 前置：第 01-02 章

★ 范式转变：从「即问即答」到「深思熟虑」

传统 LLM 是即问即答：输入问题，立刻输出答案，中间没有显式思考。这像考试时「凭直觉作答」。推理模型引入了一个新维度：推理时计算（Test-time Compute）——在给答案前，先生成一段长长的思维过程。

传统 LLM：问题 → 答案（快，但易错）
vs
推理模型：问题 → [几百~几千 token 的思考] → 答案（慢，但准）
图 17-1 · 推理模型的核心区别：先思考后作答

这个看似简单的改变，带来了数学/代码/科学推理能力的飞跃。背后的洞察是：

「更多的思考时间 = 更好的答案」。人不也是这样吗？难题多想想，正确率更高。推理模型把这个直觉用 RL 系统化地教给了模型。

01 思维链：推理的基石

Chain-of-Thought（CoT，思维链）是一切的起点。2022 年 Google 发现：在 prompt 里加一句「Let's think step by step」（让我们一步步思考），模型就会生成中间推理步骤，准确率大幅提升。

CoT 的效果对比
# 不用 CoT（直接答）
问：一个商店有 23 个苹果，卖了 17 个，又进了 12 个，现在有几个？
答：18 个。  # ❌ 错误！正确是 18... 凑巧对了，换题就错

# 用 CoT（一步步思考）
问：一个商店有 23 个苹果，卖了 17 个，又进了 12 个，现在有几个？让我们一步步思考。
答：原有 23 个，卖了 17 个，剩下 23-17=6 个；
    又进了 12 个，现在 6+12=18 个。
    答案：18 个。  # ✅ 正确，且过程可验证

CoT 为什么有效

  • 分解复杂问题：把一个大问题拆成多个小步骤，每步更简单；
  • 更多计算 = 更深推理：生成更多 token 意味着模型做了更多层前向计算；
  • 中间结果可复用：前一步的结果作为后一步的输入，类似「工作记忆」；
  • 可验证：推理过程暴露出来，便于检查错误。

CoT 的演进：从 prompt 到训练

阶段方式代表
CoT promptingprompt 里要求一步步想原始 CoT 论文
Zero-shot CoT加「Let's think step by step」Kojima et al.
Self-Consistency多采样 + 投票第 5 章第 11 节
训练型推理用 RL 让模型天生会推理o1 / R1 / GLM 推理

关键转折：o1 和 R1 不再依赖 prompt 触发 CoT，而是把推理能力训练进模型本身。它们会自动生成长思维链，不需要你在 prompt 里说「一步步想」。这是「推理模型」与传统 LLM 的本质区别。

02 RLVR：推理模型的训练引擎

推理模型的核心训练方法是 RLVR（Reinforcement Learning with Verifiable Rewards，可验证奖励的强化学习）。第 2 章第 15 节给了代码雏形，这里深入原理。

为什么 RLVR 对推理特别有效

传统 RLHF 依赖人类标注偏好，昂贵且主观。RLVR 的洞察：对数学、代码这类有客观标准答案的任务，可以完全跳过人类标注：

数学题："求 x²+3x-10=0 的解"
▼ 模型生成多个推理路径
路径A → x=2 ✅
路径B → x=-5 ✅
路径C → x=3 ❌
▼ 验证器自动判对错（标准答案: x=2 或 x=-5）
奖励：A=1, B=1, C=0
▼ 强化学习更新
模型学会：正确路径的概率↑，错误路径↓
图 17-2 · RLVR：用可验证的正确性作奖励，无需人类标注

关键：奖励信号是客观、自动、无限的。模型可以自我博弈：不断生成解法、验证、改进，循环数百万次，推理能力螺旋上升。这就是 R1 能以极低成本训出顶级推理能力的原因。

RLVR 的训练目标

对每个数学/代码问题 q：
1. 采样 N 条推理轨迹 {τ₁...τₙ}
2. 用验证器判断每条对错：r(τᵢ) ∈ {0, 1}
3. 用 GRPO/PPO 更新：最大化 Σ r(τᵢ)·log π(τᵢ|q)

DeepSeek 的 GRPO（Group Relative Policy Optimization）是 R1 的核心算法——它不需要单独的 critic 网络，而是用一组采样的相对奖励估计 baseline，比 PPO 更省显存、更稳定。

RLVR 涌现的「Aha 时刻」

R1 训练中最迷人的现象：模型自发学会了「反思」。训练到某个阶段，模型开始在思维链里出现这样的内容：

🔑 R1 论文里的「Aha moment」

模型思维链中出现：「Wait, let me reconsider this...」（等等，让我重新想想）——它会主动检查自己的推理、发现错误、自我纠正。这种行为不是被显式训练的，而是在 RL 过程中涌现的。这是 AI 推理能力的里程碑时刻。

03 推理时计算：Scaling 的新维度

传统 Scaling Law 关注「训练算力 vs 性能」。推理模型开辟了新维度：推理算力 vs 性能。

推理算力 ↑（生成更多思考 token）
▼
准确率 ↑（尤其在难题上）
图 17-3 · 推理时计算 Scaling：想得越久，对得越多

这意味着：同一个推理模型，给更多「思考预算」就能解决更难的问题。这是与传统模型最大的不同——传统模型推理算力固定（只输出一个答案），推理模型可以动态调整思考量。

难度自适应：根据题目难度分配思考量

理想情况下，模型应该简单题少想、难题多想。实现方式：

  • 早停：模型对简单题快速收敛，提前结束思考；
  • 预算控制：API 层面限制最大思考 token；
  • 路由：先用小模型判断难度，难题才调推理模型。

这是 2026 年成本优化的重点——推理模型很贵（生成大量思考 token），必须按需使用。

04 什么时候该用推理模型

场景推荐原因
复杂数学证明✅ 推理模型需要多步推理，准确率差异巨大
困难算法题✅ 推理模型需要规划+试错
科学研究分析✅ 推理模型需要深度推理
日常闲聊❌ 普通模型推理模型太慢太贵，无必要
简单信息查询❌ 普通模型不需要推理
创意写作❌ 普通模型推理模型可能过于「理性」
代码生成（简单）❌ 普通模型快模型够用
代码生成（复杂架构）✅ 推理模型需要规划
⚠️ 推理模型不是万能的
  • 慢：生成几千思考 token，延迟几秒到几十秒；
  • 贵：token 消耗是普通模型的 5-50 倍；
  • 对简单任务过度思考：可能把简单问题复杂化；
  • 不一定擅长开放任务：没有标准答案的创意任务，推理优势不明显。

05 代码实战：调用推理模型

reasoning_demo.py —— 调用推理模型并观察思维链
"""
reasoning_demo.py —— 调用推理模型，观察其思维链
"""
from zhipuai import ZhipuAI
client = ZhipuAI()

resp = client.chat.completions.create(
    model="glm-z1-plus",        # 推理模型
    messages=[{"role": "user", "content": "证明 √2 是无理数"}],
    stream=True,
)
# 推理模型会先输出「思考过程」，再给最终答案
for chunk in resp:
    delta = chunk.choices[0].delta
    # 部分推理模型把思考放在单独字段
    if hasattr(delta, "reasoning_content") and delta.reasoning_content:
        print(f"\033[90m[思考] {delta.reasoning_content}\033[0m", end="")
    elif delta.content:
        print(f"\033[32m{delta.content}\033[0m", end="")
# 输出：先看到长长的思考过程（灰色），再看到答案（绿色）

06 本章小结

📌 核心要点
  • 推理模型 = 先生成思维链，再给答案，用更多推理时计算换准确率；
  • CoT 是基石，o1/R1 把推理能力训练进模型（不再靠 prompt）；
  • RLVR（可验证奖励 RL）是训练引擎——数学/代码有标准答案，无需人类标注；
  • GRPO 让 RL 更省更稳；训练中涌现「自我反思」；
  • 推理时计算是 Scaling 新维度：想得越久对得越多；
  • 适合难题，不适合简单任务（慢且贵）。
推理模型代表了 AI 从「快思考」走向「慢思考」的进化。它和第 5 章的智能体结合——智能体负责行动，推理模型负责深思——就是最强大的 AI 形态。
