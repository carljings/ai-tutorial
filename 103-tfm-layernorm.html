层归一化
LayerNorm、RMSNorm 与 Pre-Norm 之争

深层 Transformer 有个大麻烦：梯度会爆炸或消失，让训练根本起不来。归一化（Normalization）是解药——它把每层的激活值「拉回」到稳定的范围。原论文用 LayerNorm，现代大模型（Llama、GLM）几乎全换成了更省的 RMSNorm。本章讲清为什么不用 BatchNorm、LayerNorm 和 RMSNorm 的数学差异、Pre-Norm vs Post-Norm 的训练稳定性之争，以及归一化位置对深层网络的关键影响。

阅读约 26 分钟 难度 ★★★★ 前置：第 102 章

★ 为什么需要归一化

先看不做归一化会怎样。一个几十层的 Transformer，每层都是矩阵乘 + 激活。如果每层的输出尺度稍微偏大一点（比如放大 1.1 倍），经过 96 层后，1.1^96 ≈ 9000 倍——激活值爆炸成天文数字，softmax 全饱和，梯度消失或爆炸，训练彻底崩。

深层网络的灾难：每层的小扰动，经过指数放大，让激活尺度失控。要么爆炸（NaN），要么消失（梯度为 0）。训练根本起不来。

归一化的作用：在每层（或每个子层）之后，把激活值强行拉回均值 0、方差 1 的范围，再通过可学习的缩放（γ）和偏移（β）恢复到合适尺度。这样无论前面多少层，每层拿到的输入都是「规格化」的，尺度可控。

每层输出可能尺度漂移
▼ 归一化
拉回均值 0、方差 1，再用 γ、β 调整
▼ 传给下一层
下一层拿到「干净」的输入
▼ 多层累积不失控
深层网络可训练

01 为什么不用 BatchNorm

在 CV（卷积网络）里，BatchNorm 是王者。它对一个 batch 内、同一个特征通道的所有样本做归一化——也就是「跨样本、按特征」统计均值方差。为什么 Transformer 不用它？

  • batch 维度在 NLP 里有问题：句子长度不一，batch 里短句要 padding，padding 部分会污染 batch 统计。而且不同 batch 的统计量差异大，不稳定。
  • 推理时依赖 batch 统计：BatchNorm 推理时用训练累积的移动平均统计，这在 batch=1（单条推理）或 batch 分布变化时会有 train/serve skew。
  • 跨样本归一化语义不对：NLP 里每个序列是独立的，不该让样本 A 的统计去影响样本 B 的归一化。

LayerNorm 的思路完全不同：对单个样本、跨所有特征维度做归一化，不碰 batch 维。这样每个序列独立归一化，不受 batch 内其他样本影响，推理和训练行为一致。

对比BatchNormLayerNorm
归一化维度跨 batch、按特征跨特征、按样本
统计来源整个 batch单个样本自己
推理行为用移动平均（train/serve skew）和训练一致
序列长度敏感性敏感（padding 干扰）不敏感
适合任务CV（图像）NLP（序列）

02 LayerNorm 的数学

LayerNorm 对单个样本的特征向量 x = [x_1, x_2, ..., x_d] 做归一化：

μ = (1/d) Σ x_i     （特征维上的均值）
σ² = (1/d) Σ (x_i − μ)²     （特征维上的方差）
x̂_i = (x_i − μ) / √(σ² + ε)     （标准化）
y_i = γ_i · x̂_i + β_i     （可学习仿射变换）

这里 ε 是个小常数（如 1e-5）防止除零。γ 和 β 是可学习参数，让模型能恢复任意想要的尺度和偏移。关键：μ 和 σ 是对每个样本的 d 个特征算的，和 batch 无关。

layernorm.py —— 手写 LayerNorm
import torch
import torch.nn as nn

class LayerNorm(nn.Module):
    def __init__(self, d_model, eps=1e-5):
        super().__init__()
        self.gamma = nn.Parameter(torch.ones(d_model))  # 可学习缩放
        self.beta = nn.Parameter(torch.zeros(d_model))  # 可学习偏移
        self.eps = eps

    def forward(self, x):
        # x: (batch, seq, d_model)
        mean = x.mean(dim=-1, keepdim=True)      # 沿特征维求均值
        var = x.var(dim=-1, keepdim=True, unbiased=False)  # 沿特征维求方差
        x_norm = (x - mean) / torch.sqrt(var + self.eps)
        return self.gamma * x_norm + self.beta

# 对比 PyTorch 内置
d_model = 64
mine = LayerNorm(d_model)
builtin = nn.LayerNorm(d_model)

# 初始化成相同参数
builtin.weight.data.copy_(mine.gamma.data)
builtin.bias.data.copy_(mine.beta.data)

x = torch.randn(2, 10, d_model)
print("和内置差异:", (mine(x) - builtin(x)).abs().max().item())  # ~1e-7

LayerNorm 的两个开销：①算均值和方差（一次 reduce）；②维护 γ 和 β 两组参数（每个隐藏维度一对）。对大模型（d_model=4096、几十层），这点参数不算多，但计算和访存开销在深层里有累积。

03 RMSNorm：去掉均值和偏移

2019 年论文《Root Mean Square Layer Normalization》提出了 RMSNorm。它观察到一个现象：LayerNorm 里的「减均值」和「偏移 β」其实作用不大，真正起作用的是「除以标准差」这个尺度归一化。于是它做了个简化：

RMSNorm(x)_i = γ_i · x_i / √( (1/d) Σ x_j² + ε )

也就是：不减均值、不加偏移、只用 RMS（均方根）做归一化

RMS（Root Mean Square）= √(mean(x²))，它和标准差的区别是不减均值。RMSNorm 的三个简化：

  • 不减均值：省了一次 reduce（求和）。
  • 不加 β：参数少一半（只有 γ）。
  • 不算方差，算 RMS：省了「先求均值再求方差」的两次 reduce，只需一次平方求和。

实验证明：RMSNorm 效果和 LayerNorm 几乎一样，但计算量少 7~64%（论文数据）。Llama、GLM、Qwen、Mistral 全部采用 RMSNorm，LayerNorm 在大模型里基本被淘汰。

rmsnorm.py —— RMSNorm 实现
import torch
import torch.nn as nn

class RMSNorm(nn.Module):
    def __init__(self, d_model, eps=1e-6):
        super().__init__()
        self.gamma = nn.Parameter(torch.ones(d_model))  # 只有缩放，没有偏移
        self.eps = eps

    def forward(self, x):
        # RMS = sqrt(mean(x^2))
        rms = torch.sqrt(x.pow(2).mean(dim=-1, keepdim=True) + self.eps)
        return self.gamma * (x / rms)

# 对比参数量
d = 4096
ln = nn.LayerNorm(d)
rn = RMSNorm(d)
print(f"LayerNorm 参数: {sum(p.numel() for p in ln.parameters())}")  # 8192 (γ+β)
print(f"RMSNorm 参数:  {sum(p.numel() for p in rn.parameters())}")   # 4096 (只有γ)

# 验证：RMSNorm 不改变方向，只缩放尺度
x = torch.randn(2, 10, d)
y = rn(x)
ratio = (y / x).mean().item()
print(f"平均缩放比例: {ratio:.3f}")  # 大致是 γ / rms，每个位置不同

💡 为什么减均值不重要

Transformer 里的激活向量，经过大量矩阵乘后，均值本来就接近 0（对称的权重初始化 + 残差连接让正负大致抵消）。所以「减均值」几乎不减什么。真正影响训练稳定性的是尺度（方差/RMS），这才是必须控制的。RMSNorm 抓住了主要矛盾，丢掉了冗余操作。

05 Pre-Norm vs Post-Norm：放哪儿

归一化在 Transformer block 里放的位置，是个影响深远的决策。两种主流放法：

Post-Norm（原论文）：归一化在残差之后。

h = LayerNorm(x + Sublayer(x))
out = LayerNorm(h + Sublayer2(h))

Pre-Norm（现代主流）：归一化在残差之前，作用于子层的输入。

h = x + Sublayer(LayerNorm(x))
out = h + Sublayer2(LayerNorm(h))
Post-Norm
x + Sub(x) → LN
Pre-Norm
x + Sub(LN(x))
区别：LN 在残差外还是内
影响：梯度路径、训练稳定性
图 103-2 · Post-Norm（左）vs Pre-Norm（右）的结构差异

两者的核心差异在残差路径的完整性：

  • Post-Norm：残差 x + Sub(x) 先合并，再被 LN 归一化。残差路径被打断——梯度要穿过 LN，深层时仍有消失风险。训练需要 learning rate warmup 才能稳。
  • Pre-Norm：x 直接加到输出，残差路径是干净的高速公路，梯度能无损地一路传到深层。训练极稳定，不需要 warmup。

代价：Pre-Norm 在效果上略逊于 Post-Norm（最终性能稍低一点），因为归一化作用在子层输入上，子层看到的总是「规格化」的输入，表达力略受限。但训练稳定性带来的好处远大于这点损失——尤其对超大模型（几十亿、上百亿参数），Post-Norm 根本训不起来。所以现代大模型全部用 Pre-Norm。

经验：Post-Norm 适合小模型（< 1 亿参数），追求极致效果；Pre-Norm 是大模型（> 1 亿）的必选，否则训练崩。

06 Sandwich Norm 和其他变体

还有几种归一化位置的变体：

  • Sandwich Norm：子层前后都加 Norm（x + Sub2(LN(Sub1(LN(x))))）。更稳定但计算多。
  • DeepNorm：Microsoft 提出，针对 Post-Norm 的改进，通过特殊初始化让深层 Post-Norm 也能训。效果据说比 Pre-Norm 略好。
  • ReZero：完全不用 Norm，给每个子层一个可学习的「残差缩放」α，初始为 0。让深层网络从「恒等映射」开始慢慢学。

这些大多是小众探索，主流是 Pre-RMSNorm（Pre-Norm + RMSNorm），Llama/GLM/Qwen/Mistral 全用这个组合。简单、稳定、够好。

07 进阶：QK-Norm 和注意力里的归一化

2024 年有些大模型（如 Llama 3 的某些变体、Stable Diffusion 3）在注意力的 Q、K 上也加了 RMSNorm，叫 QK-Norm。原因是：深层大模型里 Q·K^T 的分数容易爆炸（即使有 √d_k 缩放），导致 softmax 饱和。给 Q 和 K 各加一个 RMSNorm，把它们的尺度压住，训练更稳。

qk_norm.py —— 带 QK-Norm 的注意力
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class AttentionWithQKNorm(nn.Module):
    def __init__(self, d_model, n_heads):
        super().__init__()
        self.n_heads = n_heads
        self.d_k = d_model // n_heads
        self.q_proj = nn.Linear(d_model, d_model, bias=False)
        self.k_proj = nn.Linear(d_model, d_model, bias=False)
        self.v_proj = nn.Linear(d_model, d_model, bias=False)
        # QK-Norm：每头一个 RMSNorm（按 head_dim 归一化）
        self.q_norm = RMSNorm(self.d_k)
        self.k_norm = RMSNorm(self.d_k)

    def forward(self, x):
        B, S, _ = x.shape
        q = self.q_proj(x).view(B, S, self.n_heads, self.d_k).transpose(1, 2)
        k = self.k_proj(x).view(B, S, self.n_heads, self.d_k).transpose(1, 2)
        # 关键：投影后、点积前，对 Q 和 K 各做 RMSNorm
        q = self.q_norm(q)
        k = self.k_norm(k)
        v = self.v_proj(x).view(B, S, self.n_heads, self.d_k).transpose(1, 2)
        scores = q @ k.transpose(-2, -1) / math.sqrt(self.d_k)
        return F.softmax(scores, dim=-1) @ v

# 验证 QK-Norm 把分数压在合理范围
torch.manual_seed(0)
attn = AttentionWithQKNorm(64, 8)
x = torch.randn(1, 100, 64) * 10  # 故意放大输入
q = attn.q_proj(x).view(1, 100, 8, 8)
print(f"Q 投影后尺度: std={q.std():.2f}")
q_n = attn.q_norm(q)
print(f"QK-Norm 后尺度: std={q_n.std():.2f}")  # 被 RMSNorm 压回 1 附近

QK-Norm 在超大规模（70B+）模型上尤其有用，能显著提升训练稳定性。它让深层 Transformer 在高 bf16 精度下也不容易 NaN。

08 本章小结

归一化的核心作用
  • 防止深层激活尺度漂移、梯度爆炸/消失
  • 不用 BatchNorm：NLP 里跨样本归一化不合理
  • LayerNorm：跨特征归一化，逐样本独立
  • RMSNorm：简化版，不减均值、不加 β
现代大模型的选择
  • Pre-Norm + RMSNorm 是标配
  • Pre-Norm 残差路径干净，训练稳
  • RMSNorm 省计算，效果和 LayerNorm 一致
  • QK-Norm 进一步稳定超深大模型

归一化讲完了。接下来回到注意力的核心——softmax。前面所有章节的注意力都用标准 softmax，它的 O(n²) 复杂度是长上下文的硬瓶颈。Flash Attention 用工程手段绕开了显存问题，但计算量还是 O(n²)。有没有办法从数学上把注意力复杂度降到 O(n) 甚至更低？这就是下一章的线性注意力、稀疏注意力、Latent Attention等 softmax 变体要回答的问题。
