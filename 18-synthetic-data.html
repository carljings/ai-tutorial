合成数据工程
「2026 年不可妥协」的训练燃料

人类数据快被用完了。合成数据——用 AI 生成的训练数据——已成为大模型训练不可或缺的燃料。本章深入数据飞轮、质量过滤流水线、Model Collapse 防护、以及「AI 训 AI」如何避免退化。这是理解 GLM-5 这类模型数据工程的关键。

阅读约 35 分钟 难度 ★★★★ 前置：第 02 章

★ 合成数据的必然性

Towards AI 宣称「2026 是合成数据变得不可妥协之年」。三个驱动因素：

🏜️

人类数据耗尽

高质量互联网文本基本被训练过，新增量有限。

⚖️

合规收紧

版权、隐私法规让真实数据使用受限。

📈

需求爆炸

模型越来越大，需要的数据越来越多。

Gartner 预测：2026 年 75% 企业会用生成式 AI 创建合成数据（几年前不到 5%），到 2030 年合成数据将超越真实数据成为 AI 训练主力。

01 合成数据的五大类型

类型怎么生成用途
指令数据让强模型生成「问题+答案」对SFT 训练（如 Self-Instruct）
偏好数据生成多个答案，让模型/人排序DPO/RLHF 对齐
推理轨迹生成「问题+思维链+答案」推理模型训练（第 17 章）
代码数据生成代码+测试用例代码能力训练
对话数据模拟多轮对话对话能力训练

02 数据飞轮：自我提升的循环

合成数据最强大的形态是自我提升飞轮：

强模型 M_t
▼ 生成训练数据
合成数据 D_t（生成 + 验证 + 过滤）
▼ 训练
更强的模型 M_{t+1}
▼ 生成更好的数据
合成数据 D_{t+1}（质量更高）
↺ 循环，能力螺旋上升
图 18-1 · 合成数据飞轮：AI 训 AI，越训越强

关键前提：每一步都要有质量控制，否则不是飞轮而是死亡螺旋（下一节的 Model Collapse）。

03 质量过滤：飞轮不崩的关键

合成数据最大的风险是质量参差。第 2 章第 15 节给了雏形，这里展开完整的过滤流水线：

synthetic_pipeline.py —— 完整的合成数据质量流水线
"""
synthetic_pipeline.py —— 合成数据生成 + 多层质量过滤
确保「喂给模型的每条数据都是高质量的」
"""
from zhipuai import ZhipuAI
import re
client = ZhipuAI()

# ========== ① 多样化生成 ==========
def generate_diverse(seed_topic, n=10):
    """用不同 prompt/温度生成多样化数据"""
    results = []
    styles = ["简洁", "详细", "举例说明", "对比分析", "循序渐进"]
    for i in range(n):
        style = styles[i % len(styles)]
        resp = client.chat.completions.create(model="glm-5.2", messages=[
            {"role": "system", "content": f"你是教师，用'{style}'风格讲解。"},
            {"role": "user", "content": seed_topic},
        ], temperature=0.9)
        results.append(resp.choices[0].message.content)
    return results

# ========== ② 正确性验证（可验证任务的核心） ==========
def verify_correctness(item, verifier):
    """对数学/代码等可验证任务，用执行器判断对错"""
    if item["type"] == "code":
        # 跑测试，通过=正确
        return run_tests(item["code"], item["tests"])
    elif item["type"] == "math":
        # 对比标准答案
        return extract_answer(item) == item["ground_truth"]
    return True   # 非可验证任务，留给质量模型

# ========== ③ 质量评分（LLM-as-Judge） ==========
def quality_score(item, dimensions=["准确性", "清晰度", "完整性"]):
    """用强模型给每条数据打分（1-10）"""
    resp = client.chat.completions.create(model="glm-5.2", messages=[
        {"role": "user", "content": f"给这段内容打分（{dimensions}各1-10），只输出JSON：{item}"}
    ])
    import json
    return json.loads(resp.choices[0].message.content)

# ========== ④ 去重（防同质化） ==========
def dedup(items, embedder, threshold=0.85):
    """用 embedding 相似度去重，保留信息密度高的"""
    embs = embedder.encode(items)
    kept = []
    for i, item in enumerate(items):
        is_dup = False
        for j in kept:
            if cosine(embs[i], embs[j]) > threshold:
                is_dup = True; break
        if not is_dup: kept.append(i)
    return [items[i] for i in kept]

# ========== ⑤ 难度过滤（只保留「略难于当前能力」的） ==========
def difficulty_filter(items, current_model, target_acc=0.3):
    """只保留当前模型答对率 30%-80% 的题（最有学习价值）"""
    return [it for it in items
            if target_acc < estimate_accuracy(current_model, it) < 0.8]

# ========== 完整飞轮一步 ==========
def flywheel_step(seed, current_model):
    raw      = generate_diverse(seed, n=20)           # 生成
    correct  = [x for x in raw if verify_correctness(x)]  # 验证
    high_q   = [x for x in correct if quality_score(x)["准确性"] >= 8]  # 质量分
    unique   = dedup(high_q)                          # 去重
    valuable = difficulty_filter(unique, current_model)  # 难度
    print(f"20 → {len(correct)} 正确 → {len(high_q)} 高质 → {len(unique)} 唯一 → {len(valuable)} 有价值")
    return valuable

注意漏斗式的层层过滤：20 条原始生成 → 验证正确性 → 质量评分 → 去重 → 难度筛选，最后可能只剩 2-3 条「真正有价值」的数据。合成数据的精髓不是「生成得多」，而是「过滤得精」。

04 Model Collapse：合成数据的头号风险

如果不加过滤地用 AI 生成数据训 AI，几代之后模型会退化——这就是第 9 章提过的 Model Collapse。

真实数据（分布丰富，有长尾稀有内容）
▼ 第 1 代模型
生成数据（丢失部分长尾）
▼ 第 2 代（用合成数据训练）
生成数据（更多长尾丢失，分布收窄）
▼ 多代之后
分布塌缩：只会生成最常见的模式，多样性丧失
图 18-2 · Model Collapse：合成数据反复使用导致分布退化

数学上，每一代合成都会让分布的方差缩小（mode collapse），稀有但重要的信息逐渐消失。防护措施：

防护原理
始终混入真实数据真实数据是「锚」，防止分布漂移
主动追求多样性去重 + 多样化生成 + 覆盖长尾
质量过滤丢弃低质量合成数据
跟踪分布漂移监控合成数据的分布是否在收窄
人类在环路关键数据由人审核
⚠️ Model Collapse 的现实威胁

互联网上已经有越来越多 AI 生成的内容。下一代的爬虫爬到的数据里，AI 生成内容占比越来越高。如果不加区分地用这些数据训练，模型会悄然退化。这也是为什么顶级团队极其重视数据溯源——区分真实人类内容和 AI 生成内容，优先用前者。

05 高级合成技术

🔄

Self-Instruct

让模型自己生成「问题」，自己回答，构造指令数据。

⚔️

Evol-Instruct

逐步增加问题难度（简单→复杂→专家级），生成梯度数据。

🎭

Persona 驱动

给模型不同「人格」生成多样化视角的数据。

🔬

回译/改写

翻译/改写真实数据，扩充规模同时保持质量。

虽然 GLM-5.2 独立技术报告未公开，但基于行业实践和 GLM-5 报告，可推断其合成数据策略包含：

  • 推理轨迹：用推理模型生成高质量思维链数据；

  • 难度课程学习：按难度排序，从易到难训练。

★ 实战：用 LLM 生成合成训练数据的完整 pipeline

前面讲了 Self-Instruct、Evol-Instruct 等技术，这里把它们串成一个可运行的 pipeline：用强模型生成指令-回答对，再用规则+模型做质量过滤与去重，最终产出可用于 SFT 的数据集。这是 Phi、Qwen 等模型快速提升能力的核心工程。

synthetic_pipeline.py · Self-Instruct + 质量过滤 + 去重
"""合成 SFT 数据 pipeline：种子 → 扩增 → 过滤 → 去重 → 导出。"""
import json, random, hashlib
from openai import OpenAI

client = OpenAI(base_url="https://open.bigmodel.cn/api/paas/v4", api_key="YOUR_KEY")
GEN_MODEL, JUDGE_MODEL = "glm-4-plus", "glm-4-plus"

SEED_TOPICS = ["Python 异步编程", "SQL 窗口函数", "正则表达式"]

# ① Self-Instruct：从种子话题生成多样化指令
def gen_instructions(topic, n=5):
    prompt = (f"围绕主题『{topic}』生成 {n} 个互不相同的编程问题，"
              "覆盖从入门到进阶难度，每行一个，只要问题本身。")
    r = client.chat.completions.create(model=GEN_MODEL, messages=[
        {"role": "user", "content": prompt}], temperature=0.9)
    return [q.strip() for q in r.choices[0].message.content.splitlines() if q.strip()]

# ② 生成回答
def gen_answer(question):
    r = client.chat.completions.create(model=GEN_MODEL, messages=[
        {"role": "system", "content": "你是资深工程师，给出准确、带代码示例的回答。"},
        {"role": "user", "content": question}], temperature=0.7)
    return r.choices[0].message.content

# ③ 质量过滤：用模型当裁判打分（拒绝低质）
def quality_ok(question, answer):
    r = client.chat.completions.create(model=JUDGE_MODEL, messages=[
        {"role": "user", "content": f"判断回答质量。问题：{question}

回答：{answer}

"
         "只输出 1（合格，准确且有代码）或 0（不合格）。"}], temperature=0.0)
    return "1" in r.choices[0].message.content[:3]

# ④ 去重：按问题归一化哈希去重（防分布塌缩）
def dedup_key(question):
    norm = "".join(sorted(question.replace(" ", "").lower()))
    return hashlib.md5(norm.encode()).hexdigest()

if __name__ == "__main__":
    seen, dataset = set(), []
    for topic in SEED_TOPICS:
        for q in gen_instructions(topic):
            if dedup_key(q) in seen:        # 跳过重复问题
                continue
            ans = gen_answer(q)
            if not quality_ok(q, ans):      # 质量过滤
                continue
            seen.add(dedup_key(q))
            dataset.append({"instruction": q, "output": ans})
            print(f"✓ 保留 ({len(dataset)}): {q[:40]}")

    with open("synthetic_sft.jsonl", "w", encoding="utf-8") as f:
        for d in dataset:
            f.write(json.dumps(d, ensure_ascii=False) + "
")
    print(f"产出 {len(dataset)} 条 SFT 数据 → synthetic_sft.jsonl")
# 关键防护：① 高温扩增多样性 ② 裁判过滤保质量 ③ 去重防塌缩
# 进阶：再混入真实数据做锚、用 Evol-Instruct 提升难度梯度

这个 pipeline 把本章的所有要点编码进了工程：高温生成追求多样性、裁判模型保质量、哈希去重防分布塌缩。真正的生产级 pipeline 还会加上难度课程排序（先易后难训练）、拒绝采样（生成多个解法只留对的，RLVR 的基础）、以及始终混入真实数据作为锚——这些都是防止 Model Collapse 的关键。

★ 合成数据质量检测

合成数据做完，最难的不是生成而是"怎么证明它有用"。好的合成数据要"统计上像真实、又不能泄露真实样本"。下面这个函数从三个维度量化合成质量：

import numpy as np
from scipy.stats import wasserstein_distance
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import cross_val_score

def synthetic_quality(real: np.ndarray, synth: np.ndarray) -> dict:
    """评估合成数据相对真实数据的质量，三个维度：
    1. 边际分布相似度 (Wasserstein 距离，越小越好)
    2. 列间相关性保持 (相关系数矩阵差的 Frobenius 范数)
    3. 区分难度 (分类器能否分辨真/假，准确率越接近 0.5 越好)
    """
    report = {}
    n_feat = real.shape[1]

    # 1. 每列的 Wasserstein 距离，取平均
    wd = np.mean([wasserstein_distance(real[:, j], synth[:, j])
                  for j in range(n_feat)])
    report["marginal_wasserstein"] = round(float(wd), 4)

    # 2. 相关性结构保持
    corr_real = np.corrcoef(real, rowvar=False)
    corr_synth = np.corrcoef(synth, rowvar=False)
    report["corr_frobenius"] = round(
        float(np.linalg.norm(corr_real - corr_synth)), 4)

    # 3. 区分难度：标签 0=real, 1=synth，用分类器交叉验证准确率
    X = np.vstack([real, synth])
    y = np.array([0]*len(real) + [1]*len(synth))
    clf = LogisticRegression(max_iter=500)
    acc = cross_val_score(clf, X, y, cv=5, scoring="accuracy").mean()
    report["discriminator_acc"] = round(float(acc), 4)
    report["quality_score"] = round(
        (1 - wd) * (1 - abs(acc - 0.5) * 2), 4)  # 越接近 1 越好
    return report

# 演示：好的合成 vs 差的合成
rng = np.random.default_rng(0)
real = rng.normal(0, 1, (1000, 5))
good_synth = rng.normal(0, 1.02, (1000, 5))    # 分布接近
bad_synth = rng.normal(1.5, 2.0, (1000, 5))    # 分布偏离

print("好的合成:", synthetic_quality(real, good_synth))
print("差的合成:", synthetic_quality(real, bad_synth))
# discriminator_acc 接近 0.5 = 分类器分不清真假 = 合成质量高
# 这是评估 GAN/扩散生成数据最实用的单一指标

07 本章小结

📌 核心要点
  • 合成数据是 2026 训练燃料，驱动因素：数据耗尽/合规/需求；
  • 飞轮：强模型→好数据→更强模型，核心是质量控制；
  • 质量过滤是漏斗：生成→验证→评分→去重→难度，层层筛选；
  • Model Collapse是头号风险，靠混真实数据+多样性+过滤防护；
  • RLVR（第 17 章）让可验证任务的合成数据可以全自动质量控制。
合成数据工程是当代 AI 团队的核心壁垒。GLM-5 敢用 27 万亿 token 训练，其中合成数据的比例和质量管控是关键。掌握它，就掌握了「让模型持续进化」的钥匙。
